   - `pub fn config_path(app_handle: &tauri::AppHandle) -> Result<std::path::PathBuf, ConfigError>` â€” uses `app_handle.path().app_data_dir()` then joins `"config.json"`. The `app_data_dir()` method requires importing `tauri::Manager` trait.
   - `pub fn load_or_create_config(app_handle: &tauri::AppHandle) -> Result<AppConfig, ConfigError>` â€” if file exists, read + deserialize; if not, create default + save + return
   - `pub fn save_config(app_handle: &tauri::AppHandle, config: &AppConfig) -> Result<(), ConfigError>` â€” create_dir_all on parent, serialize pretty, write
   - `pub fn detect_repo_info(path: &str) -> Result<RepoInfo, ConfigError>` â€” open with `git2::Repository::open()`, extract dir name as repo name, list local branches, detect merge_target (prefer "develop" then "main" then first branch)

2. Create `src-tauri/src/commands/config_commands.rs` with these Tauri commands:

   - `get_config(app_handle: tauri::AppHandle) -> Result<AppConfig, ConfigError>` â€” calls `load_or_create_config`
   - `add_project(app_handle: tauri::AppHandle, path: String) -> Result<AppConfig, ConfigError>`:
     - Validate path exists (PathNotFound error if not)
     - Call `detect_repo_info(&path)` to get name, merge_target, branches
     - Check not already registered (compare paths, case-insensitive on Windows)
     - Generate UUID v4 for project id
     - Create ProjectConfig with detected defaults (name from dir, merge_target from detect, branch_prefix "wt/", empty build_files, None changelog)
     - Add to config.projects, save, return full config
   - `remove_project(app_handle: tauri::AppHandle, id: String) -> Result<AppConfig, ConfigError>`:
     - Find project by id, error if not found
     - Remove from projects vec, save, return config
   - `update_project(app_handle: tauri::AppHandle, id: String, name: Option<String>, merge_target: Option<String>, branch_prefix: Option<String>, build_files: Option<Vec<BuildFileConfig>>, changelog: Option<Option<ChangelogConfig>>) -> Result<AppConfig, ConfigError>`:
     - Find project by id, apply non-None updates, save, return config
     - Note: `changelog: Option<Option<ChangelogConfig>>` â€” outer Option means "field provided", inner Option means "set or clear"
   - `update_settings(app_handle: tauri::AppHandle, refresh_interval: Option<u32>, start_minimized: Option<bool>, start_with_windows: Option<bool>) -> Result<AppConfig, ConfigError>`:
     - Apply non-None updates to settings, save, return config
   - `check_project_health(path: String, merge_target: String) -> Result<HealthStatus, ConfigError>`:
     - Call `check_health(&path, &merge_target)`, return result

   Use `#[tauri::command]` on each function. All return `Result<T, ConfigError>`.

   Concurrency safety: Use `tauri::State<std::sync::Mutex<()>>` as a write lock parameter on all mutating commands (add, remove, update_project, update_settings). This prevents concurrent writes from corrupting the config file. Initialize the mutex in lib.rs via `.manage(std::sync::Mutex::new(()))`.

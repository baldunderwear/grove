**3. Create `src-tauri/src/process/launch.rs`:**
- Import `std::process::Command` and `std::os::windows::process::CommandExt`
- Define `const CREATE_NEW_CONSOLE: u32 = 0x00000010;`
- Implement `launch_claude_session(worktree_path: &str, worktree_name: &str, extra_flags: &[String]) -> Result<u32, String>`:
  - Try `wt.exe` first: `wt.exe -d <worktree_path> --title "Claude: <worktree_name>" claude <extra_flags>`
  - On `NotFound` error, fall back to `cmd.exe /c start "Claude: <name>" cmd /k "cd /d <worktree_path> && claude <flags>"` with `creation_flags(CREATE_NEW_CONSOLE)`
  - Return the child PID (note: for wt.exe this is wt's PID, not claude's -- that's expected per Pitfall 3)
- Implement `launch_vscode(worktree_path: &str) -> Result<(), String>`:
  - `Command::new("code").arg(worktree_path).spawn()` -- fire and forget
  - Map spawn error to descriptive string

**4. Create `src-tauri/src/process/detect.rs`:**
- Import `sysinfo::System` and `std::collections::HashMap`
- Define `pub struct SessionDetector { system: System }`
- Implement `SessionDetector::new() -> Self` with `System::new()`
- Implement `detect_active_sessions(&mut self, worktree_paths: &[String]) -> HashMap<String, u32>`:
  - Call `self.system.refresh_processes(sysinfo::ProcessesToUpdate::All, true)`
  - Iterate all processes. For each process:
    - Get process name via `process.name().to_string_lossy()`
    - Skip if name does not contain "claude" (case-insensitive)
    - Also check `process.cmd()` first few args for "claude" (handles Node.js wrapper case per Pitfall 5)
    - For matching processes, normalize cwd and cmd args paths: `replace('\\', "/").to_lowercase()`
    - Match against each worktree_path (also normalized) via `starts_with`
    - Insert `worktree_path -> process.pid().as_u32()` into result map
  - Return the map

**1. Create `src-tauri/src/commands/session_commands.rs`** with 5 Tauri commands:

- `launch_session(worktree_path: String, worktree_name: String, launch_flags: Vec<String>) -> Result<u32, String>`:
  - Calls `crate::process::launch::launch_claude_session(&worktree_path, &worktree_name, &launch_flags)`
  - Returns PID on success (FR-03.1, FR-03.2)

- `get_active_sessions(worktree_paths: Vec<String>, detector: tauri::State<'_, std::sync::Mutex<crate::process::detect::SessionDetector>>) -> Result<std::collections::HashMap<String, u32>, String>`:
  - Locks the detector mutex
  - Calls `detector.detect_active_sessions(&worktree_paths)`
  - Returns the HashMap (FR-03.4)

- `open_in_vscode(worktree_path: String) -> Result<(), String>`:
  - Calls `crate::process::launch::launch_vscode(&worktree_path)` (FR-03.5)

- `open_in_explorer(worktree_path: String, app: tauri::AppHandle) -> Result<(), String>`:
  - Uses `app.opener().reveal_item_in_dir(&worktree_path)` from tauri-plugin-opener (FR-03.5)
  - Import `tauri_plugin_opener::OpenerExt` for the `.opener()` method on AppHandle

- `create_worktree(project_path: String, branch_name: String, branch_prefix: String) -> Result<String, crate::git::error::GitError>` (FR-03.3):
  - Open repo via `git2::Repository::open(&project_path)`
  - Construct full branch name: `format!("{}{}", branch_prefix, branch_name)`
  - Determine worktree path: parent directory of project_path joined with the full branch name
  - Get HEAD commit: `repo.head()?.peel_to_commit()?`
  - Create the worktree via `repo.worktree()` with `WorktreeAddOptions`
  - Return the worktree path as a String
  - Uses GitError for error handling (consistent with git_commands.rs pattern)

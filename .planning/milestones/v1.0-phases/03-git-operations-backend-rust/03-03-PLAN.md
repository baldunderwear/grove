    **watcher/mod.rs** - File watcher with Tauri event bridge:

    Event payload struct (derive Clone, Serialize):
    - `GitChangeEvent`: project_path (String), change_type (String -- one of "branch_added", "branch_removed", "status_changed", "refs_changed")

    **`pub fn start_watcher(app: tauri::AppHandle, project_paths: Vec<String>) -> Result<(), String>`:**
    1. If project_paths is empty, return Ok (nothing to watch)
    2. Create `std::sync::mpsc::channel()` for debounced events
    3. Try creating a debounced watcher with `notify_debouncer_mini::new_debouncer(Duration::from_secs(2), tx)`
    4. **NAS fallback (Pitfall 5):** If the recommended watcher fails to watch any path, fall back to `notify::PollWatcher` with a 5-second interval. Log the fallback.
    5. For each project_path, derive watch paths:
       - `{project_path}/.git/refs/` -- branch ref changes
       - `{project_path}/.git/worktrees/` -- worktree additions/removals (FR-06.2, FR-06.3)
       - `{project_path}/.git/HEAD` -- HEAD changes
       - Use `RecursiveMode::Recursive` for refs and worktrees dirs
       - Use `RecursiveMode::NonRecursive` for .git/ itself (HEAD file)
       - IMPORTANT: If the path is a worktree (has a .git file instead of .git dir), follow the gitdir pointer to the parent repo's .git
    6. Spawn a `std::thread::spawn` that:
       - Loops on `rx.recv()`
       - For each batch of events, filter to git-relevant paths:
         - `is_git_relevant(path)` returns true if path contains `.git/refs/`, `.git/worktrees/`, `.git/HEAD`, or `.git/index`
       - Determine change_type from path:
         - Path in `.git/worktrees/` + creation event = "branch_added"
         - Path in `.git/worktrees/` + deletion event = "branch_removed"
         - Path in `.git/refs/` = "refs_changed"
         - Otherwise = "status_changed"
       - Match the event's path back to a project_path (find which project directory is a prefix of the changed path)
       - Emit Tauri event: `app.emit("git-changed", GitChangeEvent { project_path, change_type })`
    7. Store the debouncer handle by leaking it (Box::leak) or keeping it in a background thread scope to prevent Drop from stopping the watcher. Alternative: store in a `OnceCell` or thread-local.

    **`pub fn stop_watcher()`** - For now, a no-op placeholder. The watcher runs for the app's lifetime. If we need dynamic add/remove later, this is the extension point.

    **`fn is_git_relevant(path: &std::path::Path) -> bool`:**
    Check if path string (with forward-slash normalization) contains any of: `.git/refs/`, `.git/worktrees/`, `.git/HEAD`, `.git/index`, `.git/MERGE_HEAD`.

    **`fn classify_change(path: &std::path::Path) -> &'static str`:**
    Match on path components to return "branch_added", "branch_removed", "refs_changed", or "status_changed".

    IMPORTANT: The watcher thread must hold the `app` AppHandle clone (it is Clone + Send). The debouncer and watcher must remain alive -- do not let them drop.

    2. In the `.setup(|app| { ... })` closure, AFTER the existing tray icon and window event setup, add watcher initialization:
    ```rust
    // Start file watcher for registered project paths
    let app_handle = app.handle().clone();
    // Load config to get project paths -- if config load fails, skip watcher (non-fatal)
    if let Ok(config) = crate::config::persistence::load_or_create_config(&app_handle) {
        let paths: Vec<String> = config.projects.iter().map(|p| p.path.clone()).collect();
        if !paths.is_empty() {
            // Start watcher in background -- log errors but don't block app startup
            if let Err(e) = crate::watcher::start_watcher(app_handle, paths) {
                eprintln!("Warning: file watcher failed to start: {}", e);
            }
        }
    }
    ```

    IMPORTANT: Watcher failure must NOT prevent app from starting. Wrap in if-let/match and log errors. The dashboard still works via manual refresh without the watcher.

    IMPORTANT: The `app.handle().clone()` must be used (not `app` itself) because `app` is borrowed by the setup closure. AppHandle is Clone + Send.

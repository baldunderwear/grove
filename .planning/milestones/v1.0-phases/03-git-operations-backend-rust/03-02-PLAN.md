    **git/build.rs** - Build number detection and bumping:

    - `pub fn detect_current_build(repo_path: &str, build_patterns: &[BuildFileConfig]) -> Result<Option<u32>, GitError>`:
      1. For each pattern in build_patterns, use `glob::glob(&format!("{}/{}", repo_path, pattern.pattern))` to find matching files
      2. Read each file content as string
      3. Parse build number using `extract_build_number(&content)` (see below)
      4. Track maximum build number found across all files
      5. Return `Some(max)` if any found, `None` if no build files or no numbers found

    - `fn extract_build_number(content: &str) -> Option<u32>`:
      Support three common formats:
      1. JSON: look for `"build":\s*(\d+)` or `"build_number":\s*(\d+)` using string search (not regex crate -- keep deps minimal)
      2. Plain text: if entire file content (trimmed) is a single number, parse it
      3. TOML-style: look for `build = (\d+)` or `build_number = (\d+)`
      Use simple string scanning (find the key, parse the following digits). No regex crate needed.

    - `pub fn bump_build_number(repo_path: &str, build_patterns: &[BuildFileConfig], new_build: u32) -> Result<Vec<String>, GitError>`:
      1. For each matching file from glob
      2. Read content, find build number location
      3. Replace old number with new_build
      4. Write updated content back
      5. Return list of modified file paths (relative to repo_path, normalized with forward slashes)

    - `fn replace_build_number(content: &str, new_build: u32) -> String`:
      Same detection logic as extract, but replaces the found number with the new value.

    **git/changelog.rs** - Changelog fragment operations:

    - `ChangelogFragment` struct (derive Serialize, Clone): path (String), name (String), is_legacy (bool)

    - `pub fn find_changelog_fragments(repo_path: &str, changelog_config: &ChangelogConfig, worktree_name: &str) -> Result<Vec<ChangelogFragment>, GitError>`:
      1. Build the changelog directory path: `{repo_path}/{changelog_config.directory}`

    - `pub fn rename_changelog_fragments(repo_path: &str, changelog_config: &ChangelogConfig, worktree_name: &str, new_build: u32) -> Result<Vec<(String, String)>, GitError>`:
      1. Find fragments using find_changelog_fragments
      2. For standard fragment (worktree-{name}.md): rename to `{new_build}.md` in same directory
      3. For legacy numbered fragments: leave them (they're from old merges, already named correctly)
      4. Return list of (old_path, new_path) tuples for reporting
      5. Use `std::fs::rename` for the actual file move

    **git/merge.rs** - Merge preview and execution:

    Data types (all derive Serialize, Clone):
    - `MergePreview`: source_branch (String), target_branch (String), commits_to_merge (Vec<CommitInfo>), changelog_fragments (Vec<ChangelogFragment>), current_build (Option<u32>), next_build (Option<u32>), can_fast_forward (bool), has_conflicts (bool)
    - `CommitInfo`: oid (String), message (String), author (String), timestamp (i64)
    - `MergeResult`: success (bool), new_build (Option<u32>), commits_merged (usize), changelog_renames (Vec<(String, String)>), warnings (Vec<String>)
    - `ConflictInfo`: path (String), is_build_file (bool)

    **`pub fn merge_preview(project_path: &str, source_branch: &str, merge_target: &str, build_patterns: &[BuildFileConfig], changelog_config: &Option<ChangelogConfig>) -> Result<MergePreview, GitError>`:**
    1. Open repo, find source and target branch OIDs
    2. Run `repo.merge_analysis(&[&annotated_commit])` to check fast-forward possibility
    3. Use revwalk from source to target to list commits being merged (walk source, hide target)
    4. For each commit: extract oid (hex), message, author name, timestamp into CommitInfo
    5. Detect current build number via `build::detect_current_build`
    6. Compute next_build = current_build.map(|n| n + 1)
    7. If changelog_config is Some: find fragments via `changelog::find_changelog_fragments` (need worktree name -- extract from source_branch by stripping branch_prefix)
    8. Check for conflicts: `repo.merge_commits(&our_commit, &their_commit, None)`, then `index.has_conflicts()`
    9. Return MergePreview (this is read-only, no mutations)

    **`pub fn merge_branch(project_path: &str, source_branch: &str, merge_target: &str, build_patterns: &[BuildFileConfig], changelog_config: &Option<ChangelogConfig>) -> Result<MergeResult, GitError>`:**
    Atomic merge transaction:
    1. Open repo, resolve source and target commits
    2. Save rollback point: `let original_head_oid = repo.head()?.target().unwrap()`
    3. Make sure HEAD is on merge_target: `repo.set_head(&format!("refs/heads/{}", merge_target))`
    4. Perform `repo.merge_commits(&our_commit, &their_commit, None)` to get merged index
    5. Check conflicts:
       - If `merged_index.has_conflicts()`:
         - Iterate conflicts, classify each as build_file or not using `is_build_file_conflict(&path, build_patterns)` (glob match)
         - If ANY non-build conflicts: return `Err(GitError::UnexpectedConflict(paths))` -- HEAD unchanged, safe
         - For build-file-only conflicts: auto-resolve by removing conflict entries from index, adding "ours" version via `index.add_frombuffer()` with the target's blob content
    6. Write merged tree: `merged_index.write_tree_to(&repo)`
    7. If build_patterns not empty: bump build number on disk files, then update the tree to include bumped files
       - Read bumped files from disk, add to index via repo, write new tree
    8. If changelog_config is Some: rename fragments on disk, update tree similarly
    9. Get commit signature: try `repo.signature()`, fall back to `Signature::now("Grove", "grove@localhost")` (Pitfall 6)
    10. Create merge commit with 2 parents: `repo.commit(Some("HEAD"), &sig, &sig, &message, &final_tree, &[&our_commit, &their_commit])`
       - Message format: "Merge {source_branch} into {merge_target}" with optional "Build: {new_build}" line
    11. Checkout HEAD to update working directory: `repo.checkout_head(Some(CheckoutBuilder::default().force()))`
    12. Return MergeResult with success=true, counts, and warnings

    CRITICAL: merge is LOCAL only. No `repo.remote()` or push calls anywhere (FR-04.8).

    Helper: `fn is_build_file_conflict(path: &str, build_patterns: &[BuildFileConfig]) -> bool` -- use `glob::Pattern::new(&pattern.pattern)` and match against the path.

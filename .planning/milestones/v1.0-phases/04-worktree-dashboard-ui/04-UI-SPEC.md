## Layout Contract

### Overall Structure

```
+------------------+---------------------------------------------+
|                  |  Dashboard Header                           |
|  Sidebar (280px) |  [Project Name]    [Sort: v] [Refresh btn]  |
|  (Phase 02,      |  [branch count] [last refreshed timestamp]  |
|   unchanged)     |--------------------------------------------|
|                  |  Branch Table                               |
|                  |  [header row]                               |
|                  |  [branch row 1]                             |
|                  |  [branch row 2]                             |
|                  |  [branch row ...]                           |
|                  |  (scrollable)                               |
+------------------+---------------------------------------------+
```

### Dashboard Header

- Top: Project name (20px semibold, gray-50) left-aligned
- Below project name: "{N} branches" count (12px, gray-400) + separator dot + "Updated {relative time}" (12px, gray-400)
- Right side of header: Sort dropdown + Manual refresh button (icon-only, lucide `RefreshCw`, 20px)
- Refresh button spins its icon while a refresh is in progress (CSS animation, 1s rotation)
- Header bottom border: gray-800 separator

### Sort Controls

- Dropdown trigger shows current sort: "Activity" (default), "Name", "Commits ahead"
- Dropdown trigger: ghost button variant, 14px, gray-400 text, lucide `ArrowUpDown` icon (16px) left of label
- Dropdown menu: 3 items, each with a check mark for active sort
- Sort is client-side only (no backend call)

### Branch Table

Table layout with the following columns:

| Column | Width | Content |
|--------|-------|---------|
| Status | 40px fixed | Status dot: clean (emerald-500), dirty (amber-500), or stale (gray-500) |
| Branch | flex-1 (min 200px) | Branch name (14px, gray-50, truncate) + last commit message below (12px, gray-400, truncate, 1 line) |
| Ahead/Behind | 120px fixed | "+{N}" in emerald-400 if > 0, gray-500 if 0. "-{N}" in gray-500. Separated by "/" |
| Last Activity | 140px fixed | Relative timestamp ("3h ago"), 14px, gray-400. Stale branches (> 7 days): gray-500 + lucide `Clock` icon (14px) |
| Badges | auto | Horizontal badge row: "Merge ready" (emerald), "Dirty" (amber), "Stale" (gray), "Active" (emerald pulse) |

### Table Row Behavior

- Default: transparent background, gray-800 bottom border
- Hover: gray-850 (#1e1e1e) background
- Row height: 48px minimum (auto-expand for two-line content)
- No row selection state in this phase (selection comes in Phase 05 with launch actions)
- Entire row is not clickable in this phase (individual actions come in Phase 05/06)

### Branch Table Header Row

- Background: gray-900
- Text: 12px, uppercase, letter-spacing 0.05em, gray-400, weight 400
- Sticky at top of scroll area
- Columns: (blank for status) | Branch | Ahead / Behind | Last Activity | (blank for badges)

### Empty State (No Branches)

Shown when project is selected but has zero worktree branches matching the prefix.

- Centered vertically and horizontally in main content area
- Lucide `GitBranch` icon, 48px, gray-500
- Heading: "No worktree branches" (20px semibold, gray-300)
- Body: "No branches matching prefix \"{prefix}\" found. Create a worktree to get started." (14px, gray-400)
- No CTA button in this phase (worktree creation is Phase 05)

### Loading State

- On initial load: 6 skeleton rows in table body. Each skeleton row has: circle (status dot), rectangle (branch name), small rectangle (ahead/behind), small rectangle (timestamp)
- On refresh: subtle overlay -- existing data stays visible, refresh icon spins, rows have 50% opacity during fetch (200ms transition)
- Skeleton color: gray-800 with shimmer animation (gray-700 sweep, 1.5s duration, infinite)

### Badge Specifications

| Badge | Background | Text Color | Border | Content |
|-------|-----------|------------|--------|---------|
| Merge ready | emerald-500/15 | emerald-500 | none | "Ready" |
| Dirty | amber-500/15 | amber-500 | none | "Dirty" |
| Stale | gray-500/15 | gray-500 | none | "Stale" |
| Active session | emerald-500/15 | emerald-500 | none | lucide `Terminal` 12px + "Active" |

Badge sizing: 12px text, 4px vertical padding, 8px horizontal padding, rounded-full.

Active session badge has a pulsing emerald-500 dot (6px, CSS animation: opacity 0.4-1.0, 2s ease-in-out infinite) before the text.

## Interactions

| Action | Behavior |
|--------|----------|
| Click project in sidebar | Load branches via `list_branches` Tauri command, show dashboard |
| Click sort dropdown item | Re-sort branch list client-side, no API call |
| Click refresh button | Call `list_branches` + check dirty status per branch. Spin icon during fetch. |
| Hover branch row | Background changes to gray-850 |
| Hover truncated branch name | Tooltip shows full branch name |
| File watcher event received | Auto-refresh branch list (same as manual refresh but without icon spin) |
| Auto-refresh timer fires | Silent refresh: fetch new data, diff with current, update changed rows only. No loading overlay. |
| Window regains focus | Trigger immediate refresh if last refresh was > 10s ago |

## State Inventory

| State | Visual Treatment |
|-------|-----------------|
| Initial load (no data yet) | 6 skeleton rows in table, refresh icon spinning |
| Branches loaded, all healthy | Table populated, sort active, auto-refresh running |
| Branches loaded, some stale | Stale branches show gray badge + clock icon, sorted below active by default |
| Branches loaded, some dirty | Dirty branches show amber "Dirty" badge |
| Branches loaded, some merge-ready | Merge-ready branches show emerald "Ready" badge (ahead > 0, not dirty) |
| Branches loaded, active sessions | Active branches show pulsing "Active" badge (placeholder -- actual detection in Phase 05) |
| No branches match prefix | Empty state with GitBranch icon |
| Refresh in progress | Existing rows at 50% opacity, refresh icon spinning |
| Auto-refresh in progress | No visible indicator (silent background update) |
| Error: fetch failed | Error message in gray-400 text where table would be, with "Try again" button |
| Project path unavailable | Error state: "Project path is not accessible. Check if the drive is connected." |

## Data Flow

```
sidebar click -> selectProject(id)
             -> invoke("list_branches", { path, prefix, target })
             -> for each branch: invoke("is_worktree_dirty", { worktree_path })
             -> setState({ branches, loading: false })
             -> render table

auto-refresh timer -> same flow, but no loading overlay
file watcher event -> same flow, but no loading overlay
manual refresh btn -> same flow, with spinning icon
```

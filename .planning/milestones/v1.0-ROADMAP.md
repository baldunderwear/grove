## Overview

Grove v1.0 delivers a complete Windows desktop worktree manager for Claude Code. Built with Tauri 2 + React 19 + TypeScript, it provides a system tray resident app for managing git worktrees, launching Claude Code sessions, and performing merge operations with automatic build number bumping and changelog management.

## Milestone Summary

**Key Decisions:**
- NAS workaround: node_modules on local C: drive via junction (scripts/with-modules.mjs)
- Tailwind v4 via @tailwindcss/vite plugin (no PostCSS config)
- Tray-resident pattern: window starts hidden, close hides to tray
- shadcn/ui with zinc dark theme, emerald-500 accent
- Separate Zustand stores per domain (config, branch, session, merge)
- Git CLI for fetch (inherits SSH credentials), git2 for everything else
- Transition-only notifications (HashMap state tracking prevents spam)

**Issues Resolved:**
- Phase 07: start_minimized setting not reading config on startup (fixed inline)
- Phase 07: merge-completed notification not wired (fixed inline)
- Phase 08: Ctrl+N keyboard shortcut not reaching NewWorktreeDialog (fixed inline)
- Integration: tray launch-worktree event had no frontend listener (fixed during audit)
- Integration: tray navigate event had no frontend listener (fixed during audit)
- Integration: tray menu not refreshing after project add/remove (fixed during audit)

**Technical Debt Incurred:**
- 3 orphaned Tauri commands (branch_status, is_worktree_dirty, resolve_build_conflicts)
- Launch flags always empty (no UI to configure per-launch flags)
- File watcher not restarted for newly added projects (relies on timer refresh)
- Theme setting saved but has no effect (dark mode hardcoded)
- UPDATER_PUBKEY_PLACEHOLDER needs replacement before first release
- grove:close-dialog custom event is dead code (radix handles Escape natively)

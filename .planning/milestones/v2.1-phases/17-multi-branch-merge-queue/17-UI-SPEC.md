# Phase 17 — UI Design Contract

> Visual and interaction contract for the multi-branch merge queue: branch selection via checkboxes, drag-reorder dialog, sequential merge execution with progress feedback, and in-place toast updates. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 13px | 600 (semibold) | 1.3 |
| Heading | 14px | 600 (semibold) | 1.3 |
| Mono | 12px | 400 (regular) | 1.4 |

The 12/13/14px scale is intentional continuity from the Phase 14 and Phase 15 design system, which established these sizes for Grove's dense information layout. This phase does not introduce new sizes; it reuses the existing scale to maintain visual consistency across all dashboard surfaces.

Heading is used for the dialog title. Label is used for queue item branch names and progress status text. Mono is used for commit counts and build numbers in queue items.

## Focal Point

Primary visual anchor: "Merge Selected ({N})" button in BranchTable header when 2+ branches are checked. This is the entry point to the entire merge queue flow and must be immediately visible against the table header background. Inside the dialog, the focal point shifts to the "Start Queue" primary button in the footer.

## MergeQueueDialog Layout

### Dialog Shell

| Property | Value |
|----------|-------|
| Width | 480px (`max-w-[480px]`) |
| Background | `var(--grove-deep)` with `border border-[var(--grove-canopy)]` |
| Border radius | 6px (`var(--radius)`) |
| Overlay | Default shadcn dialog overlay (semi-transparent black) |

### Dialog Sections (top to bottom)

1. **Header**: Dialog title "Merge Queue" at Heading (14px/600), description "{N} branches selected" at Body (14px/400) in `var(--grove-stone)`.

2. **Queue List** (pre-execution): Sortable list of branch items inside a `scroll-area` with max-height 320px. Each `<QueueItem />`:
   - Left: 6-dot drag handle icon (`GripVertical` lucide, 14px) in `var(--grove-stone)`, cursor grab
   - Center: branch name at Label (13px/600) in `var(--grove-fog)`, below it "+{N} commits" at Mono (12px/400) in `var(--grove-stone)`
   - Right: remove button (X icon, 14px) in `var(--grove-stone)`, visible on hover, with `aria-label="Remove {branch-name} from queue"` for screen reader accessibility
   - Background: `var(--grove-forest)` with `border border-[var(--grove-canopy)]`, rounded-md (4px)
   - Spacing: 8px padding inside item, 8px gap between items
   - Drag active state: item gets `ring-1 ring-[var(--grove-moss)]` and slight scale `scale-[1.02]` with `shadow-lg`

3. **Queue List** (during execution): Same layout, but drag handles hidden, remove buttons hidden. Each item shows status:
   - Pending: `Circle` icon (lucide, 14px) in `var(--grove-stone)`
   - Active: `Loader2` icon (lucide, 14px, `animate-spin`) in `var(--grove-leaf)`
   - Complete: `CheckCircle2` icon (lucide, 14px) in `emerald-400`
   - Failed: `XCircle` icon (lucide, 14px) in `red-500`
   - Rolled back: `RotateCcw` icon (lucide, 14px) in `amber-400`

4. **Progress Section** (during execution only): Below queue list, above footer.
   - Progress bar: full width, 4px height, `var(--grove-canopy)` track, `var(--grove-leaf)` fill
   - Status text: "Merging {current}/{total}: {branch-name}" at Label (13px/600) in `var(--grove-fog)`, centered below bar with 8px top gap

5. **Footer**:
   - Pre-execution: "Discard Queue" ghost button (left), "Start Queue" primary button (right) with `bg-[var(--grove-leaf)] hover:bg-[var(--grove-sprout)] text-[var(--grove-void)]`
   - During execution: "Start Queue" replaced with disabled "Merging..." button with `Loader2` spinner; "Discard Queue" hidden (queue cannot be interrupted mid-execution)
   - After success: "Done" primary button (right), closes dialog
   - After failure: "Close" ghost button (right), closes dialog

## Interaction Contract

### Branch Selection Flow

1. User clicks existing "Select" toggle in BranchTable header (existing `showSelection` prop) to reveal checkboxes
2. User checks 2 or more branches that have `ahead > 0` (merge-ready)
3. "Merge Selected ({N})" button appears in BranchTable header row, right-aligned, at `sm` button size
4. Button uses `bg-[var(--grove-leaf)] hover:bg-[var(--grove-sprout)] text-[var(--grove-void)]` with `GitMerge` icon (14px) left of label
5. Button is disabled if fewer than 2 branches are selected
6. Clicking the button opens `<MergeQueueDialog />` with the selected branches in their current table sort order

### Drag Reorder

1. User can grab any queue item by its drag handle (6-dot grip icon) and drag to reorder
2. Drop indicator: 2px horizontal line in `var(--grove-leaf)` at the insertion point
3. Reordering is only possible before queue execution starts
4. After reorder, the queue store updates the branch order
5. Keyboard reorder: focus drag handle, press Space to pick up, Arrow keys to move, Space to drop, Escape to cancel

### Queue Execution

1. User clicks "Start Queue" to begin sequential merge execution
2. Dialog transitions to execution mode: drag handles hidden, remove buttons hidden, progress section appears
3. Rust backend receives ordered branch list, records HEAD OID snapshot, begins sequential merges
4. Frontend receives Tauri events for each branch status update: `merge-queue-progress` with `{ index, total, branch, status }`
5. Queue items update in real-time: pending -> active -> complete/failed
6. Progress bar advances: width = `(completed / total) * 100%`
7. Dialog cannot be closed during execution (escape key and overlay click disabled)

### Toast Updates During Execution (TOAST-05)

1. When queue starts: `toast.loading("Merging 1/{total}: {branch}", { id: 'merge-queue' })` creates a persistent progress toast
2. On each branch completion: `toast.loading("Merging {n}/{total}: {branch}", { id: 'merge-queue' })` updates in-place
3. On queue success: `toast.success("Queue complete: {total}/{total} merged", { id: 'merge-queue', duration: 5000 })` replaces progress toast
4. On queue failure: `toast.error("Queue failed on {branch} ({n}/{total}). Rolled back.", { id: 'merge-queue', duration: Infinity })` replaces progress toast
5. Toast uses stable ID `'merge-queue'` so Sonner updates in-place rather than spawning new toasts

### Rollback Behavior

1. If any branch fails to merge, Rust backend performs `git reset --hard` to the pre-queue HEAD OID snapshot
2. All previously completed items in the queue list change to "Rolled back" state (amber `RotateCcw` icon)
3. The failed item shows "Failed" state (red `XCircle` icon)
4. Remaining pending items stay in "Pending" state
5. Progress bar fill changes to red-500 at the failure point
6. Dialog shows "Close" button (not "Done") after failure

### File Watcher Suppression

1. Before queue execution: Rust sets `queue_active = true` flag in managed state
2. File watcher checks flag and skips refresh events while true
3. After queue completes (success or failure + rollback): Rust sets `queue_active = false` and triggers one forced refresh
4. Frontend receives the refresh event and updates branch data

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (BranchTable) | "Merge Selected ({N})" (opens queue dialog) |
| Primary CTA (Dialog) | "Start Queue" (begins execution) |
| Dialog title | "Merge Queue" |
| Dialog description | "{N} branches selected" |
| Dialog dismiss (pre-execution) | "Discard Queue" (ghost button, closes dialog without executing) |
| Progress status | "Merging {current}/{total}: {branch-name}" |
| Success status | "Queue complete: {total}/{total} merged" |
| Failure status | "Queue failed on {branch-name} ({current}/{total}). Rolled back." |
| Empty state | Not applicable (dialog only opens with 2+ branches selected) |
| Error state (queue failure in dialog) | "Merge failed on {branch-name}. All {completed} previous merges have been rolled back." |
| Error state (queue failure toast) | "Queue failed on {branch-name} ({current}/{total}). Rolled back." |
| Toast progress | "Merging {current}/{total}: {branch-name}" |
| Toast success | "Queue complete: {total}/{total} merged" |
| Destructive confirmation | Not applicable (rollback is automatic, not user-initiated; queue start is the deliberate action) |
| Remove branch from queue | No confirmation needed (non-destructive, branch remains in table) |

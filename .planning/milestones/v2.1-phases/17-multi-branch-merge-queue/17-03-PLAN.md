**Imports** (follow project convention order):
- React: `useState`
- lucide-react: `GripVertical`, `Circle`, `Loader2`, `CheckCircle2`, `XCircle`, `RotateCcw`, `X`
- @dnd-kit/react: `DragDropProvider`
- @dnd-kit/react/sortable: `useSortable`, `isSortable`
- shadcn: `Button`, `Dialog`, `DialogContent`, `DialogHeader`, `DialogTitle`, `DialogDescription`, `DialogFooter`, `ScrollArea`
- stores: `useMergeQueueStore`
- types: `QueueBranch`, `QueueStep`

**Structure (5 sections per UI-SPEC):**

1. **Header:** Dialog title "Merge Queue", description "{N} branches selected" using `var(--grove-stone)`.

2. **Queue List (pre-execution, step === 'ready'):**
   - Wrap items in `<DragDropProvider>` with `onDragEnd` handler.
   - `onDragEnd` handler: if canceled return. Check `isSortable(operation.source)`. Use `source.sortable.initialIndex` and `source.sortable.index` for reorder. Do NOT use `target` (known 0.3.x issue per research). Call `reorder(initialIndex, index)`.
   - Each item is a `SortableQueueItem` using render-props pattern:
     ```tsx
     function SortableQueueItem({ id, index, disabled, children }: {
       id: string; index: number; disabled: boolean;
       children: (args: { containerRef: (el: Element | null) => void; handleRef: (el: Element | null) => void; isDragging: boolean }) => React.ReactNode;
     }) {
       const { ref, handleRef, isDragging } = useSortable({ id, index, disabled });
       return <>{children({ containerRef: ref, handleRef, isDragging })}</>;
     }
     ```
   - Item layout per UI-SPEC: left=GripVertical drag handle (ref=handleRef, cursor-grab), center=branch name (13px/600) + "+N commits" below (12px/400), right=X remove button (hover visible).
   - Item styling: `bg-[var(--grove-forest)] border border-[var(--grove-canopy)] rounded-md p-2 gap-2`.
   - Drag active: `ring-1 ring-[var(--grove-moss)] scale-[1.02] shadow-lg`.
   - Items gap: `space-y-2` or `gap-2` flex column.

3. **Queue List (during execution, step === 'executing' | 'success' | 'failure'):**
   - Same list but drag disabled (`disabled` prop on SortableQueueItem), remove buttons hidden.
   - Status icon per item based on `branch.status`:
     - `pending`: `<Circle className="h-3.5 w-3.5 text-[var(--grove-stone)]" />`
     - `active`: `<Loader2 className="h-3.5 w-3.5 animate-spin text-[var(--grove-leaf)]" />`
     - `complete`: `<CheckCircle2 className="h-3.5 w-3.5 text-emerald-400" />`
     - `failed`: `<XCircle className="h-3.5 w-3.5 text-red-500" />`
     - `rolled_back`: `<RotateCcw className="h-3.5 w-3.5 text-amber-400" />`

4. **Progress Section (execution only):**
   - Below queue list, visible when step is 'executing'.
   - Progress bar: div with `h-1 rounded-full bg-[var(--grove-canopy)]` track, inner div with `bg-[var(--grove-leaf)]` fill at `width: ${(completed/total)*100}%`. If any failure, fill becomes `bg-red-500`.
   - Status text: "Merging {current}/{total}: {branch-name}" at 13px/600 in `text-[var(--grove-fog)]` centered below with `mt-2`.
   - Compute completed count: `branches.filter(b => b.status === 'complete').length`.
   - Current branch: `branches[currentIndex]`.

5. **Footer:**
   - Pre-execution (step === 'ready'): "Discard Queue" ghost button (onClick: reset + onOpenChange(false)), "Start Queue" primary button (bg-[var(--grove-leaf)] hover:bg-[var(--grove-sprout)] text-[var(--grove-void)]).
   - During execution (step === 'executing'): Single disabled button "Merging..." with Loader2 spinner.
   - After success (step === 'success'): "Done" primary button (closes dialog and resets).
   - After failure (step === 'failure'): "Close" ghost button (closes dialog and resets).

**Dialog behavior:**
- Width: `max-w-[480px]`.
- Background: `bg-[var(--grove-deep)] border border-[var(--grove-canopy)]`.
- During execution: prevent close via `onOpenChange` -- ignore close attempts when step === 'executing'. Set `onInteractOutside={(e) => { if (step === 'executing') e.preventDefault(); }}` on DialogContent.
- On close (when allowed): call `reset()` from store.

**Start Queue handler:**
```typescript
const handleStartQueue = () => {
  startQueue(projectPath, mergeTarget, buildPatterns, changelogConfig);
};
```

**Key anti-patterns to avoid (from research):**
- Do NOT use `target` in onDragEnd -- use `source.sortable.initialIndex` and `source.sortable.index`.
- Do NOT apply `ref` to the drag handle -- `ref` (containerRef) goes on the outer div, `handleRef` goes on the GripVertical icon.
- Do NOT allow drag during execution -- pass `disabled={step !== 'ready'}` to SortableQueueItem.

Export the component as a named export.

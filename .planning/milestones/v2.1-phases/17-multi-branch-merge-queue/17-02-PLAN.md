**State interface:**

```typescript
interface MergeQueueState {
  branches: QueueBranch[];
  step: QueueStep;
  currentIndex: number;
  error: string | null;

  // Actions
  setBranches: (branches: QueueBranch[]) => void;
  reorder: (fromIndex: number, toIndex: number) => void;
  removeBranch: (name: string) => void;
  startQueue: (projectPath: string, mergeTarget: string, buildPatterns: BuildFileConfig[], changelogConfig: ChangelogConfig | null) => Promise<void>;
  updateProgress: (progress: QueueProgress) => void;
  reset: () => void;
}
```

**Import pattern** (follow project convention):
```typescript
import { create } from 'zustand';
import { invoke } from '@tauri-apps/api/core';
import { listen } from '@tauri-apps/api/event';
import { fireMergeQueueToast, completeMergeQueueToast, failMergeQueueToast } from '@/lib/alerts';
import type { QueueBranch, QueueProgress, QueueResult, QueueStep } from '@/types/merge';
import type { BuildFileConfig, ChangelogConfig } from '@/types/config';
```

**Action implementations:**

`setBranches(branches)`: Set branches array, set step to 'ready', reset currentIndex to 0, clear error.

`reorder(fromIndex, toIndex)`: Create a new array with the item at fromIndex moved to toIndex. Use a simple arrayMove helper (splice-based, 3 lines -- do NOT import from @dnd-kit/helpers unless it's confirmed available):
```typescript
function arrayMove<T>(arr: T[], from: number, to: number): T[] {
  const copy = [...arr];
  const [item] = copy.splice(from, 1);
  copy.splice(to, 0, item);
  return copy;
}
```
Define this helper at module scope above the store.

`removeBranch(name)`: Filter out the branch by name. If fewer than 2 remain, reset to idle.

`startQueue(projectPath, mergeTarget, buildPatterns, changelogConfig)`:
1. Set step to 'executing'.
2. Set up Tauri event listener: `const unlisten = await listen<QueueProgress>('merge-queue-progress', (event) => { ... })`.
3. In the listener callback:
   - Call `get().updateProgress(event.payload)`.
   - If `status === 'active'`, call `fireMergeQueueToast(event.payload.index + 1, event.payload.total, event.payload.branch)`.
4. Extract branch names: `const branchNames = get().branches.map(b => b.name)`.
5. Call `const result = await invoke<QueueResult>('merge_queue_execute', { projectPath, projectName: '', branches: branchNames, mergeTarget, buildPatterns, changelogConfig })`.
6. After invoke resolves:
   - If `result.success`: call `completeMergeQueueToast(result.total)`, set step to 'success'.
   - If `!result.success`: call `failMergeQueueToast(result.failed_branch!, result.completed + 1, result.total)`, set step to 'failure', set error to `result.error`.
7. Call `unlisten()` to clean up the event listener.
8. Wrap the entire flow in try/catch. On catch: set step to 'failure', set error, call `failMergeQueueToast`.

`updateProgress(progress)`: Update the branch at `progress.index` with the new status. Set `currentIndex` to `progress.index`. Use Zustand Map cloning pattern (create new branches array with the updated item).

`reset()`: Set branches to [], step to 'idle', currentIndex to 0, error to null.

**Export:**

```

**Functions:**

export function fireMergeQueueToast(current: number, total: number, branch: string) {
  toast.loading(`Merging ${current}/${total}: ${branch}`, {
    id: MERGE_QUEUE_TOAST_ID,
  });
}

export function completeMergeQueueToast(total: number) {
  toast.success(`Queue complete: ${total}/${total} merged`, {
    id: MERGE_QUEUE_TOAST_ID,
    duration: 5000,
  });
}

export function failMergeQueueToast(branch: string, current: number, total: number) {
  toast.error(`Queue failed on ${branch} (${current}/${total}). Rolled back.`, {
    id: MERGE_QUEUE_TOAST_ID,
    duration: Infinity,
  });
}
```

The stable `id: MERGE_QUEUE_TOAST_ID` ensures Sonner updates the existing toast in-place rather than spawning new ones. `toast.loading()` shows a spinner, `toast.success()` auto-dismisses after 5s, `toast.error()` persists until manually dismissed.

Do NOT add these toasts to the `activeToasts` tracking array -- the merge queue toast is managed by its stable ID, not by the capacity system. The queue only ever has one toast at a time.

Verify the existing `toast` import from 'sonner' includes `toast.loading` -- it should since sonner ^2.0.7 supports it.

  <action>
    1. In `src-tauri/src/config/models.rs`, add a `PromptTemplate` struct following the Profile pattern:
       ```rust
       #[derive(Debug, Clone, Serialize, Deserialize)]
       pub struct PromptTemplate {
           pub id: String,
           pub name: String,
           pub body: String,           // The prompt text with {variable} placeholders
           #[serde(default)]
           pub variables: Vec<String>,  // Variable names extracted from body, e.g. ["branch", "issue"]
       }
       ```
    2. Add `templates: Vec<PromptTemplate>` field to `AppConfig` with `#[serde(default)]` for backward compat. Also add to Default impl with `templates: Vec::new()`.

    3. In `src-tauri/src/commands/config_commands.rs`, add three `#[tauri::command]` functions following the exact profile CRUD pattern:
       - `add_template(name: String, body: String, app_handle: tauri::AppHandle) -> Result<AppConfig, ConfigError>`:
         Generate UUID id via `uuid::Uuid::new_v4().to_string()`, extract variables by scanning `body` for `{word}` patterns using a regex (add `regex` crate if not present, or use simple string scanning: iterate chars, collect between `{` and `}`), create `PromptTemplate`, push to `config.templates`, save, return config.
       - `update_template(id: String, name: Option<String>, body: Option<String>, app_handle: tauri::AppHandle) -> Result<AppConfig, ConfigError>`:
         Find template by id, update name/body if Some, re-extract variables from body if body changed, save, return config.
       - `remove_template(id: String, app_handle: tauri::AppHandle) -> Result<AppConfig, ConfigError>`:
         Remove template with matching id using `retain(|t| t.id != id)`, save, return config.

       For variable extraction: use a simple loop that finds `{...}` patterns in the body string. No regex crate needed — just scan for `{`, collect until `}`, push to variables vec. Filter out empty strings. Deduplicate.

    4. Register all three commands in `src-tauri/src/lib.rs` invoke_handler, after the `set_project_profile` line:
       ```
       commands::config_commands::add_template,
       commands::config_commands::update_template,
       commands::config_commands::remove_template,
       ```

   - `read_text_file(path: String) -> Result<String, String>`: Read file at path, return contents as UTF-8 string. Return error if file not found or not readable. Cap at 512KB (return error if larger, per NFR-09 headroom).
   - `write_text_file(path: String, content: String) -> Result<(), String>`: Write content to path, creating parent dirs if needed. Validate path is not empty.
   - `list_directory(path: String) -> Result<Vec<DirEntry>, String>` where `DirEntry` is a struct with `name: String, is_dir: bool, size: u64`. List immediate children of a directory. Used by skills browser to list `.claude/skills/` contents.

3. Register all three commands in `src-tauri/src/lib.rs` invoke_handler:

   - `commands::file_commands::write_text_file`
   - `commands::file_commands::list_directory`

<task type="auto">
  <name>Task 2: Profile model, persistence, and CRUD commands</name>
  <files>src-tauri/src/config/models.rs, src-tauri/src/config/persistence.rs, src-tauri/src/commands/config_commands.rs, src-tauri/src/lib.rs, src-ui/src/types/config.ts</files>
  <read_first>src-tauri/src/config/models.rs, src-tauri/src/config/persistence.rs, src-tauri/src/commands/config_commands.rs, src-ui/src/types/config.ts</read_first>
  <action>
1. Add to `src-tauri/src/config/models.rs`:
   ```rust
   #[derive(Debug, Clone, Serialize, Deserialize)]
   pub struct Profile {
       pub id: String,
       pub name: String,
       #[serde(default)]
       pub claude_config_dir: Option<String>,
       #[serde(default)]
       pub env_vars: std::collections::HashMap<String, String>,
       #[serde(default)]
       pub ssh_key: Option<String>,
       #[serde(default)]
       pub launch_flags: Vec<String>,
       #[serde(default)]
       pub is_default: bool,
   }
   ```

2. Add to `AppConfig`:
   - `#[serde(default)] pub profiles: Vec<Profile>` field

3. Update `AppConfig::default()` to include `profiles: Vec::new()`.

4. Add to `ProjectConfig`:
   - `#[serde(default, skip_serializing_if = "Option::is_none")] pub profile_id: Option<String>` field

5. Add four Tauri commands in `src-tauri/src/commands/config_commands.rs`:
   - `add_profile(app_handle, name: String) -> Result<AppConfig, ConfigError>`: Creates profile with generated UUID id, given name, is_default=false (unless it's the first profile, then is_default=true per PROF-05). Saves config.
   - `update_profile(app_handle, id: String, name: Option<String>, claude_config_dir: Option<Option<String>>, env_vars: Option<HashMap<String, String>>, ssh_key: Option<Option<String>>, launch_flags: Option<Vec<String>>, is_default: Option<bool>) -> Result<AppConfig, ConfigError>`: Updates fields if Some. If is_default=Some(true), clear is_default on all other profiles first.
   - `remove_profile(app_handle, id: String) -> Result<AppConfig, ConfigError>`: Remove profile by id. Clear profile_id on any projects referencing it. If removed profile was is_default and others remain, set first remaining as default.
   - `set_project_profile(app_handle, project_id: String, profile_id: Option<String>) -> Result<AppConfig, ConfigError>`: Set or clear the profile_id on a project.

6. Register all four in `src-tauri/src/lib.rs` invoke_handler.

7. Update `src-ui/src/types/config.ts`:
   - Add `Profile` interface: `{ id: string; name: string; claude_config_dir: string | null; env_vars: Record<string, string>; ssh_key: string | null; launch_flags: string[]; is_default: boolean; }`
   - Add `profiles: Profile[]` to `AppConfig`
   - Add `profile_id: string | null` to `ProjectConfig`
  </action>

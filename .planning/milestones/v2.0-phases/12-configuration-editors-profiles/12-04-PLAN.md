1. Create `src-ui/src/components/config/ProfileEditor.tsx`:
   - Component manages profile list + selected profile editing.
   - **Profile list** (left panel, ~30% width):
     - Lists all profiles from `useConfigStore(s => s.config?.profiles)`.
     - Each item shows profile name, a "(default)" badge if `is_default`, and click-to-select.
     - "New Profile" button at top: prompts for name (inline input + Enter to confirm), calls `addProfile(name)`.
     - Selected profile highlighted with `--grove-canopy` background.
   - **Profile form** (right panel, ~70% width):
     - Fields for selected profile:
       - **Name**: text input, blur saves via `updateProfile(id, { name })`.
       - **Claude Config Directory**: text input with a "Browse" button using `import { open } from '@tauri-apps/plugin-dialog'` to pick a directory. Value = `claude_config_dir`. Optional — null means default.
       - **SSH Key Path**: text input with "Browse" button (file picker). Value = `ssh_key`. Optional.
       - **Environment Variables**: key-value editor. Render as rows of two inputs (key, value). "Add Variable" button appends empty row. "X" button removes row. On blur of any row, serialize all rows to `Record<string, string>` and call `updateProfile(id, { env_vars })`.
       - **Launch Flags**: tag-style input. Show existing flags as pills with "X" to remove. Input + Enter to add. On change, call `updateProfile(id, { launch_flags })`.
       - **Default Profile**: checkbox. When checked, calls `updateProfile(id, { is_default: true })` (Rust side clears other defaults).
     - **Delete Profile** button at bottom with red styling and confirmation dialog ("Delete profile '{name}'? Projects using this profile will revert to default.").
   - **Empty state**: No profile selected — "Select a profile or create a new one".
   - All form fields use Grove dark styling: `--grove-deep` backgrounds, `--grove-fog` text, `--grove-leaf` accents.

2. Create `src-ui/src/components/config/ProfileSelector.tsx`:
   - Props: none (reads from config store).
   - Renders a compact dropdown/select in the sidebar.
   - Shows current default profile name (or "No profiles" if empty).
   - Dropdown lists all profiles. Selecting one does NOT change the default — it's just for the sidebar display/filtering context. (The actual profile assignment happens per-project in ProfileEditor or ProjectConfig.)
   - Style: compact, fits in sidebar between project list and settings button. Use `--grove-stone` text, `--grove-canopy` border.
   - If no profiles exist, render nothing (return null).

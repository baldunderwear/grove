**3. Add session-state-changed listener in Dashboard.tsx:**
Add a new Effect (Effect 6) that listens for the Tauri event:

```typescript
// Effect 6: Session state change listener
useEffect(() => {
  let cancelled = false;
  const unlistenPromise = listen<{ terminal_id: string; state: string; timestamp: number }>(
    'session-state-changed',
    (event) => {
      if (cancelled) return;
      const { terminal_id, state } = event.payload;
      setTabState(terminal_id, state as SessionState);

      // Fire desktop notification on transition to "waiting"
      if (state === 'waiting') {
        const tab = useTerminalStore.getState().tabs.get(terminal_id);
        const branchName = tab?.branchName ?? 'Unknown';
        sendNotification({
          title: 'Session Waiting for Input',
          body: `${branchName} is waiting for your input`,
        });
      }
    },
  );
  return () => {
    cancelled = true;
    unlistenPromise.then((fn) => fn());
  };
}, [setTabState]);
```

Import `sendNotification` from `@tauri-apps/plugin-notification`. Check if the package is already installed; if not, run `npm install @tauri-apps/plugin-notification` in src-ui. The Rust side already has `tauri-plugin-notification` in Cargo.toml and initialized in lib.rs.

Also extract `setTabState` from the terminal store at the top of Dashboard alongside the other terminal store selectors.

function getStatusDotClass(state: SessionState): string {
  switch (state) {

    case 'waiting':  return 'bg-amber-400';
    case 'idle':     return 'bg-zinc-500';
    case 'error':    return 'bg-red-500';

}

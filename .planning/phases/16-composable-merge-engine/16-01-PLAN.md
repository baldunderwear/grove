  <name>Task 1: Create pipeline.rs with MergeContext, MergePhase, and step functions</name>
  <files>src-tauri/src/git/pipeline.rs, src-tauri/src/git/mod.rs</files>
  <action>
Create `src-tauri/src/git/pipeline.rs` containing:

1. **MergePhase enum** (derives Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord):
   - `Init` -- no steps have run
   - `Executed` -- merge_execute() completed
   - `Bumped` -- merge_bump() completed
   - `Changelogged` -- merge_changelog() completed
   - `Committed` -- merge_commit() completed

2. **MergeContext struct** with fields (see RESEARCH.md Pattern 1 for full spec):
   - Inputs: `project_path: String`, `source_branch: String`, `merge_target: String`, `build_patterns: Vec<BuildFileConfig>`, `changelog_config: Option<ChangelogConfig>`, `override_build: Option<u32>`
   - Pipeline state: `phase: MergePhase`, `warnings: Vec<String>`
   - Populated by merge_execute: `commits_merged: usize`, `merged_tree_oid: Option<git2::Oid>`, `source_oid: Option<git2::Oid>`, `target_oid: Option<git2::Oid>`
   - Populated by merge_bump: `new_build: Option<u32>`
   - Populated by merge_changelog: `changelog_renames: Vec<(String, String)>`
   - Populated by merge_commit: `commit_oid: Option<git2::Oid>`
   - Add `MergeContext::new()` constructor and `into_result() -> MergeResult` converter

3. **merge_execute(ctx: &mut MergeContext) -> Result<(), GitError>:**
   - Validates `ctx.phase == MergePhase::Init`, returns `GitError::MergeAborted` if not
   - Opens repo with `Repository::open(&ctx.project_path)`
   - Resolves source and target branches (extract from merge.rs lines 162-176)
   - Counts commits via revwalk (extract from merge.rs lines 179-183)
   - Sets HEAD to merge_target and calls checkout_head(force) (merge.rs lines 186-187) -- this is the INITIAL checkout to the target branch
   - Performs in-memory merge, handles conflicts (extract from merge.rs lines 189-213) including `classify_conflicts` and `resolve_build_conflicts_in_index` calls
   - Writes merged tree via `merged_index.write_tree_to(&repo)` (merge.rs line 216)
   - Populates `ctx.commits_merged`, `ctx.merged_tree_oid`, `ctx.source_oid`, `ctx.target_oid`
   - Does NOT call checkout_head after tree write (Pitfall 2 from RESEARCH.md)
   - Sets `ctx.phase = MergePhase::Executed`

4. **merge_bump(ctx: &mut MergeContext) -> Result<(), GitError>:**
   - Validates `ctx.phase == MergePhase::Executed`
   - If `ctx.build_patterns.is_empty()`, set `ctx.new_build = None`, advance to `Bumped`, return Ok (skip-safe)
   - If `ctx.override_build` is Some, use that value directly (do NOT call detect_current_build)
   - Otherwise call `build::detect_current_build` and compute next = current + 1 (or 1 if None)
   - Call `build::bump_build_number` to write to disk
   - Set `ctx.new_build = Some(next)`, advance to `Bumped`

5. **merge_changelog(ctx: &mut MergeContext) -> Result<(), GitError>:**
   - Validates `ctx.phase == MergePhase::Bumped`
   - If `ctx.changelog_config.is_none()` or `ctx.new_build.is_none()`, set `ctx.changelog_renames = Vec::new()`, advance to `Changelogged`, return Ok (skip-safe)
   - Extract worktree name using the `extract_worktree_name` helper (import from merge.rs -- make it `pub(crate)`)
   - Call `changelog::rename_changelog_fragments`, catching errors as warnings (matching existing behavior in merge.rs lines 241-257)
   - Set `ctx.changelog_renames`, advance to `Changelogged`

6. **merge_commit(ctx: &mut MergeContext) -> Result<(), GitError>:**
   - Validates `ctx.phase == MergePhase::Changelogged`
   - Opens repo
   - Builds final tree incorporating disk changes (extract verbatim from merge.rs lines 260-348) -- this is the complex 80-line block that reads bumped build files and renamed changelog files from disk into a git index
   - Use `ctx.merged_tree_oid`, `ctx.new_build`, `ctx.changelog_renames`, `ctx.build_patterns` to determine what needs staging
   - Creates merge commit with 2 parents using `ctx.source_oid` and `ctx.target_oid` (extract from merge.rs lines 350-369)
   - Calls `checkout_head(force)` as the FINAL operation (merge.rs line 372)
   - Sets `ctx.commit_oid`, advance to `Committed`

**Critical extraction rules:**
- Extract logic VERBATIM from merge.rs -- do not rewrite business logic, only restructure control flow
- The `classify_conflicts` and `resolve_build_conflicts_in_index` functions stay in merge.rs (they are private helpers). Import them into pipeline.rs via `super::merge::` or make them `pub(crate)`.
- The `extract_worktree_name` function needs to be accessible from pipeline.rs -- make it `pub(crate)` in merge.rs.
- The `is_build_file_conflict` function stays in merge.rs (used only by classify_conflicts).
- Store only `git2::Oid` values in MergeContext (Copy, owned). Never store Repository, Tree, or Commit (lifetime issues -- Pitfall 1).
- Each step opens Repository::open() independently (Pitfall 1).

**In mod.rs:** Add `pub mod pipeline;` to `src-tauri/src/git/mod.rs`.

Doc comments (`///`) on all public items per project conventions.

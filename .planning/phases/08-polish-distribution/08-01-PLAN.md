**src-tauri/src/lib.rs:** Add two plugin registrations in the plugin chain, after the existing `.plugin(tauri_plugin_autostart::init(...))` call:
```rust
.plugin(tauri_plugin_updater::Builder::new().build())
.plugin(tauri_plugin_process::init())
```

**src-tauri/tauri.conf.json:** Make these changes:
1. Add `"createUpdaterArtifacts": true` inside the `"bundle"` object (after `"active": true`)
2. Add a `"plugins"` top-level key with updater configuration:
```json
"plugins": {
    "updater": {
        "pubkey": "UPDATER_PUBKEY_PLACEHOLDER",
        "endpoints": [
            "https://github.com/baldunderwear/grove/releases/latest/download/latest.json"
        ]
    }
}
```
Use `"UPDATER_PUBKEY_PLACEHOLDER"` as the pubkey value. The user will generate the real key with `npx tauri signer generate` and replace this placeholder before the first release build. This is intentional -- signing keys must never be committed.

Do NOT change the version field yet -- that happens in Plan 02.

**src-tauri/capabilities/default.json:** Add two new permissions to the `"permissions"` array:
- `"updater:default"` (grants allow-check, allow-download, allow-install, allow-download-and-install)
- `"process:allow-restart"` (needed for relaunch after update install)

## Interaction Contract

### Session Exit Detection

1. When PTY process exits, frontend receives exit event with exit code
2. `terminal-store` transitions tab state from current state to `'exited'`; stores `exitCode` on the tab
3. Tab remains visible in SessionManager card grid -- it is NOT auto-removed
4. User must explicitly dismiss (X button) or navigate to merge to remove the exited card

### Exited Session Card Layout

The card retains the same outer shell (rounded-xl, `var(--grove-deep)` background, border) as live session cards. Internal content changes:

1. **Header** (unchanged): branch name + status dot. Status dot uses `emerald-500` for clean exit, `red-500` for crashed exit. Close (X) and Expand buttons remain.
2. **Exit Banner** (replaces terminal preview top area): Full-width banner inside the card body.
   - Clean exit: `bg-emerald-500/10` background, `CheckCircle2` icon in `emerald-300`, text "Session complete" in `emerald-300` at Label size (13px/600).
   - Crashed exit: `bg-red-500/10` background, `AlertTriangle` icon in `red-300`, text "Session crashed (exit code {N})" in `red-300` at Label size (13px/600).
   - Banner height: auto, with 8px vertical padding and 16px horizontal padding.
   - Banner border-radius: 8px (rounded-lg), matching the terminal preview area it replaces.
3. **Diff Summary** (replaces terminal preview lines): Shows below the exit banner within the same area.
   - Top row: aggregate stats in a single line -- "{N} files changed, {+N} insertions, {-N} deletions, {N} commits" at Body size (14px/400). Insertions in `text-emerald-400`, deletions in `text-red-400`, other text in `var(--grove-stone)`.
   - Below: scrollable file list (max 5 visible, `scroll-area` if more). Each row: file path in Mono (12px/400) in `var(--grove-bright)`, right-aligned `+{N}` in emerald-400 and `-{N}` in red-400 at Mono size.
   - If no changes detected (0 files): show "No changes detected" in `var(--grove-stone)` at Body size, centered.
4. **Footer / Action Row** (replaces duration/state footer):
   - Left: session duration (total, from creation to exit) using existing `Clock` icon + `formatDuration`.
   - Right: "Review & Merge" button (primary action) when branch has commits ahead. Button absent when no commits ahead.

### "Review & Merge" Button Spec

| Property | Value |
|----------|-------|
| Variant | Default shadcn `button` with `bg-emerald-600 hover:bg-emerald-500 text-white` |
| Size | `sm` (shadcn small) |
| Label | "Review & Merge" |
| Icon | `GitMerge` (lucide), 14px, left of label |
| Disabled state | Never disabled; absent when no commits ahead |
| Click behavior | Opens existing `MergeDialog` pre-populated with tab's branch name and project context |

For crashed sessions, the button uses `bg-amber-600 hover:bg-amber-500` and label changes to "Review & Merge" (same label, warning color indicates caution). No separate confirmation dialog -- the color change IS the warning.

### Dismiss Behavior

1. X button on exited card removes the tab from `terminal-store` (same as closing a live session)
2. No confirmation dialog for dismissing exited sessions
3. Dismissing does NOT affect the branch or worktree

### Disconnected State (Network Blip)

1. If PTY connection drops without an exit event, show state `'disconnected'` (distinct from `'exited'`)
2. Disconnected card shows: gray banner "Session disconnected" with `WifiOff` icon in `var(--grove-stone)`
3. No diff summary shown for disconnected sessions (PTY may still be running)
4. No "Review & Merge" button on disconnected cards

### Session-to-Merge Bridge

1. Clicking "Review & Merge" calls `merge-store.fetchPreview()` with the session's branch name and project path
2. MergeDialog opens in its existing `preview` step -- no new UI needed inside MergeDialog
3. After merge completes, user returns to session cards. The exited card remains until explicitly dismissed.

### No Auto-Triggering (POST-06)

1. When a session exits, the card transitions to exited state silently -- no dialog, no modal, no wizard
2. User must actively click on the exited card or its "Review & Merge" button
3. A toast fires on session exit (existing Phase 14 toast system): "Session exited" with "View Session" action

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Review & Merge" (on exited session card, opens MergeDialog) |
| Clean exit banner | "Session complete" |
| Crashed exit banner | "Session crashed (exit code {N})" |
| Disconnected banner | "Session disconnected" |
| Diff summary line | "{N} files changed, +{N} insertions, -{N} deletions, {N} commits" |
| Empty state (no changes) | "No changes detected" |
| Error state (diff fetch fails) | "Could not load diff summary" (shown in `var(--grove-stone)`, no action -- non-blocking) |
| Toast on session exit (clean) | Title: "{branch} exited" / Description: "Session completed successfully" / Action: "View Session" |
| Toast on session exit (crash) | Title: "{branch} crashed" / Description: "Exited with code {N}" / Action: "View Session" |
| Destructive confirmation | Not applicable (no destructive actions in this phase; worktree deletion deferred to Phase 18) |

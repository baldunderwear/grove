**A. ExitBanner.tsx** -- New file in `src-ui/src/components/session/`

Props: `{ exitCode: number | null; sessionState: 'exited' | 'disconnected' }`

Renders a full-width banner inside the card body per UI spec:
- **Clean exit** (exitCode === null || exitCode === 0): `bg-emerald-500/10` background, `CheckCircle2` icon (lucide) in `text-emerald-300`, text "Session complete" in `text-emerald-300` at 13px/600 weight.
- **Crashed exit** (exitCode !== null && exitCode !== 0): `bg-red-500/10` background, `AlertTriangle` icon in `text-red-300`, text "Session crashed (exit code {N})" in `text-red-300` at 13px/600 weight.
- **Disconnected** (sessionState === 'disconnected'): `bg-zinc-500/10` background, `WifiOff` icon in `text-[var(--grove-stone)]`, text "Session disconnected" in `text-[var(--grove-stone)]` at 13px/600.

- Icon size: 16px (`h-4 w-4`), placed left of text with `gap-2`.

**B. DiffSummary.tsx** -- New file

Props: `{ worktreePath: string; branchName: string; mergeTarget: string }`

This component fetches diff data lazily (on mount) and displays it:

1. Local state: `diffData: DiffSummaryData | null`, `loading: boolean`, `error: string | null`.
2. On mount, call `invoke<DiffSummaryData>('get_branch_diff_summary', { projectPath: worktreePath, sourceBranch: branchName, mergeTarget })`. Set loading true before, false after. Store result or error.
3. **Loading state**: Show a subtle skeleton/pulse animation (2-3 lines of `bg-[var(--grove-canopy)] animate-pulse rounded h-3 w-3/4` blocks).
4. **Error state**: Show "Could not load diff summary" in `text-[var(--grove-stone)]` at Body size (14px). Non-blocking, no action.
5. **Empty state** (0 files changed): Show "No changes detected" centered in `text-[var(--grove-stone)]`.
6. **Normal state**:
   - Top row: aggregate stats in a single line using `font-mono text-xs` (IBM Plex Mono via `font-mono`): "{N} files changed" in `text-[var(--grove-stone)]`, "+{N}" in `text-emerald-400`, "-{N}" in `text-red-400`, "{N} commits" in `text-[var(--grove-stone)]`. Items separated by commas or middot.
   - Below: scrollable file list. Use shadcn `ScrollArea` with max height for ~5 visible rows. Each row is a flex row: file path in `font-mono text-xs text-[var(--grove-bright)]` (truncate left, show filename), right-aligned `+{N}` in `text-emerald-400` and `-{N}` in `text-red-400` in `font-mono text-xs`.
   - Gap between stat line and file list: 8px (`gap-2`).

**Important:** The `mergeTarget` must be determined from the project config. The component receives it as a prop -- the parent (SessionCard) looks it up from `config-store` via the tab's `projectId`.

**B. SessionManager.tsx modifications:**

1. Import `setTabExited` and `setTabDisconnected` from terminal-store (add to the existing destructured imports). Import `MergeDialog` from `@/components/MergeDialog`. Import `useMergeStore`. Import `useBranchStore` for branch lookup.

2. **Exit handler change (critical):** In the `TerminalInstance` component's `onEvent.onmessage` handler, change the `Exit` case:
   ```typescript
   case 'Exit':
     write(`\r\n\x1b[90m[Process exited: ${event.code ?? '?'}]\x1b[0m\r\n`);
     if (terminalIdRef.current) {
       setTabExited(terminalIdRef.current, event.code ?? null);
     }
     break;
   ```
   Replace the `setTabConnected(id, false)` call with `setTabExited(id, exitCode)`.

3. **Error handler change:** In the `Error` case, call `setTabDisconnected` instead of doing nothing:
   ```typescript
   case 'Error':
     write(`\r\n\x1b[31m[${event.message}]\x1b[0m\r\n`);
     if (terminalIdRef.current) {
       setTabDisconnected(terminalIdRef.current);
     }
     break;
   ```

4. **Import setTabExited and setTabDisconnected** in TerminalInstance: update the destructured useTerminalStore call to include these two new actions.

5. **MergeDialog bridge:** Add state for the merge dialog in `SessionManager`:
   ```typescript
   const [mergeTabId, setMergeTabId] = useState<string | null>(null);
   ```
   When a card's `onMerge` is called:
   - Look up the tab from `tabs.get(tabId)`
   - Look up `BranchInfo` from `useBranchStore.getState().branches.find(b => b.worktree_path === tab.worktreePath)`
   - If not found, trigger a `silentRefresh` first, then retry lookup
   - Set `mergeTabId` to the tabId
   - The MergeDialog renders with `open={mergeTabId !== null}`, `branch={branchInfo}`, `project={project}`, `onOpenChange={(open) => { if (!open) setMergeTabId(null); }}`, `onComplete={() => setMergeTabId(null)}`

6. **Render MergeDialog** at the bottom of the SessionManager JSX (after the grid/focus views), conditionally when `mergeTabId` is set:
   ```tsx
   {mergeTabId && mergeTab && mergeBranchInfo && (
     <MergeDialog
       open={true}
       onOpenChange={(open) => { if (!open) setMergeTabId(null); }}
       branch={mergeBranchInfo}
       project={project}
       onComplete={() => setMergeTabId(null)}
     />
   )}
   ```

7. **Pass onMerge to SessionCard:** Update the `<SessionCard>` in the grid to pass `onMerge={handleMerge}` where `handleMerge` sets the `mergeTabId`.

8. **Keep exited tabs in grid:** Exited tabs already stay because `closeTab` is only called on explicit X button click. No change needed here -- just verify the grid renders exited cards alongside active ones.

9. **Handle close for exited sessions:** When closing an exited session (X button), skip the `terminal_kill` call since the process already exited. Check `tab.sessionState === 'exited'` or `!tab.isConnected` before calling `terminal_kill`.

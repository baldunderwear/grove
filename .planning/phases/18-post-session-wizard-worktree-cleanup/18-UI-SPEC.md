# Phase 18 — UI Design Contract

> Visual and interaction contract for the post-session wizard: a 4-step stepper dialog (Diff Summary, Commit Review, Merge, Cleanup) that replaces the current direct MergeDialog launch from PostSessionActions. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Focal Point

Primary visual anchor: the step indicator bar at the top of the dialog. It provides persistent orientation within the 4-step flow. Within each step, the focal point is the primary action button in the footer (progresses through "Next", "Merge Branch", "Clean Up & Close").

## PostSessionWizard Layout

### Dialog Shell

| Property | Value |
|----------|-------|
| Width | 520px (`max-w-[520px]`) |
| Background | `var(--grove-deep)` with `border border-[var(--grove-canopy)]` |
| Border radius | 6px (`var(--radius)`) |
| Overlay | Default shadcn dialog overlay (semi-transparent black) |

### Dialog Sections (top to bottom)

1. **Header**: Dialog title "Post-Session Review" at Heading (14px/600), description "{branch-name} into {target-branch}" at Body (14px/400) in `var(--grove-stone)`.

2. **Step Indicator** (WizardStepper): Horizontal row of 4 numbered circles (20px diameter) connected by lines.
   - Circle: 20px wide, 20px tall, `rounded-full`, 2px border
   - Completed: `bg-[var(--grove-leaf)]` with `Check` icon (12px, white)
   - Active: `bg-[var(--grove-leaf)]` with step number (12px/600, white)
   - Upcoming: `bg-transparent border-[var(--grove-canopy)]` with step number (12px/400) in `var(--grove-stone)`
   - Connectors: 1px lines between circles, `var(--grove-leaf)` for completed, `var(--grove-canopy)` for pending
   - Step labels below circles: "Summary", "Commits", "Merge", "Cleanup" at 11px/400 in `var(--grove-stone)` (active label in `var(--grove-fog)`)
   - Full width with flex justify-between, 24px gap below header

3. **Step Content Area**: Variable content based on active step. Minimum height 200px to prevent dialog size jumping between steps. Content wraps in `scroll-area` with max-height 360px.

4. **Footer**: Left side has "Back" ghost button (hidden on step 1). Right side has step-specific primary button. Separated from content by `separator` with 16px vertical spacing.

### Step 1: Diff Summary

- Section header: "Changes" at Label (13px/600) in `var(--grove-fog)`
- Summary stats row: "{N} files changed, {N} insertions, {N} deletions" at Body (14px/400) in `var(--grove-fog)`, `var(--grove-stone)` for labels
- File list: each row shows file path at Body (14px/400) in `var(--grove-fog)`, right-aligned stat "+{N}" in emerald-400, "-{N}" in red-400, at Mono (12px/400)
- File list rows have `py-1.5 px-2` padding, alternating `bg-[var(--grove-forest)]/50` for even rows
- Max 50 files shown; if more, show "{N} more files" link at bottom in `var(--grove-stone)`
- Footer button: "Next" primary button

### Step 2: Commit Review

- Section header: "{N} Commits" at Label (13px/600) in `var(--grove-fog)`
- Commit list: each row shows:
  - Left: abbreviated hash (7 chars) at Mono (12px/400) in `var(--grove-stone)`
  - Center: commit message at Body (14px/400) in `var(--grove-fog)`, truncated with ellipsis
  - Right: relative timestamp ("2h ago") at Mono (12px/400) in `var(--grove-stone)`
- Rows have `py-1.5 px-2` padding, 1px bottom border in `var(--grove-canopy)`
- Scrollable via `scroll-area`
- Footer button: "Next" primary button

### Step 3: Merge

- Pre-merge: summary confirmation text matching existing MergeDialog confirm step pattern
  - "Merge {source} into {target}" at Body (14px/400)
  - Build number bump display (if applicable): current badge -> arrow -> next badge, same style as MergeDialog
  - Commit count: "{N} commits will be merged" at Body (14px/400) in `var(--grove-fog)`
  - Italic disclaimer: "This is a local merge only -- changes will not be pushed to remote." at 12px in `var(--grove-stone)`
- During merge: centered `Loader2` spinner (24px, `animate-spin`) in `var(--grove-sprout)` with "Merging..." at Body (14px/400)
- After success: `CheckCircle2` icon (20px) in emerald-400 with "Merge complete" at Label (13px/600) in emerald-300, result summary below
- On failure: `AlertTriangle` icon (20px) in red-400 with error message at Body (14px/400) in `text-destructive`, "Retry" and "Close" buttons
- Footer button (pre-merge): "Merge Branch" primary button with `bg-emerald-600 hover:bg-emerald-500` (matches existing MergeDialog pattern)
- Footer button (post-success): "Next" primary button (advances to cleanup)
- Dialog cannot be closed during merge execution (escape and overlay click disabled)

### Step 4: Cleanup

- Section header: "Worktree Cleanup" at Label (13px/600) in `var(--grove-fog)`
- Description: "The merge is complete. Choose what to clean up:" at Body (14px/400) in `var(--grove-stone)`
- Checkbox row 1: `Checkbox` (default checked) + "Delete worktree directory" at Body (14px/400) in `var(--grove-fog)`, path below at Mono (12px/400) in `var(--grove-stone)`
- Checkbox row 2: `Checkbox` (default checked) + "Delete local branch" at Body (14px/400) in `var(--grove-fog)`, branch name below at Mono (12px/400) in `var(--grove-stone)`
- Note: "Remote branch will not be affected." at 12px in `var(--grove-stone)`, italic
- Checkbox rows have 12px vertical gap between them, 8px gap between checkbox and label
- Footer button: "Clean Up & Close" primary button with `bg-[var(--grove-leaf)] hover:bg-[var(--grove-sprout)] text-[var(--grove-void)]`
- If both checkboxes unchecked: button changes to "Close" (skip cleanup, just close dialog)
- During cleanup: button shows `Loader2` spinner, disabled state
- After cleanup: dialog closes automatically, session tab closes via `closeTab`

## Interaction Contract

### Wizard Entry Point

1. Session exits (clean or crash) and PostSessionActions renders with `hasCommitsAhead = true`
2. User clicks "Review & Merge" button (existing button, unchanged label)
3. `<PostSessionWizard />` opens at Step 1 instead of `<MergeDialog />`
4. Diff summary data is fetched via `get_branch_diff_summary` on dialog open (same as current behavior)

### Step Navigation

1. Forward navigation via primary button in footer ("Next", "Merge Branch", "Clean Up & Close")
2. Back navigation via "Back" ghost button in footer (visible on steps 2, 3 pre-merge, 4)
3. Back button is hidden during merge execution (step 3 while merging)
4. Back button is hidden after successful merge (cannot undo merge)
5. Skip forward is not allowed -- user must proceed through each step sequentially
6. Dialog close (X button, Escape, overlay click) available on all steps except during merge execution
7. Closing dialog mid-wizard does not trigger any cleanup actions

### Merge Step Behavior

1. Step 3 reuses `useMergeStore` logic: `fetchPreview` called when entering step 3 (if not already loaded), `executeMerge` on button click
2. If merge fails: user sees error with "Retry" button (retries merge) and "Close" button (closes wizard entirely)
3. If merge succeeds: step 3 shows brief success state, footer shows "Next" to advance to cleanup
4. Conflict detection: if `has_conflicts` is true in preview, show warning banner (amber border/bg) and disable "Merge Branch" button

### Cleanup Step Behavior

1. Both checkboxes default to checked
2. "Delete worktree directory" calls `delete_worktree` Tauri command
3. "Delete local branch" calls `delete_branch` Tauri command (or combined into single command)
4. Cleanup operations run sequentially: worktree first, then branch
5. If cleanup fails: show error inline, keep dialog open, let user retry or close
6. After successful cleanup: show brief success flash (200ms green tint on dialog), then auto-close dialog
7. Auto-close session tab after dialog closes (worktree no longer exists, tab is meaningless)
8. If user unchecks both and clicks "Close": dialog closes, tab remains open

### Crash Exit Handling

1. If `exitCode !== 0` (crash), wizard still opens normally -- user may still want to review and merge
2. Step 1 diff summary shows data regardless of exit type
3. No special UI treatment inside wizard for crash vs clean exit (ExitBanner already handles this on the session card)

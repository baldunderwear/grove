Create the main wizard dialog component that orchestrates step navigation and data flow.

**Props:**
```typescript
interface PostSessionWizardProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  tab: TerminalTab;
  project: ProjectConfig;
  branchName: string;
  mergeTarget: string;
}
```

**Local state (useState, NOT Zustand -- wizard state is ephemeral per locked decision):**
- `step: 0 | 1 | 2 | 3` (current wizard step)
- `diffData: DiffSummaryData | null` (fetched once on open)
- `loading: boolean` (diff data loading)
- `mergeComplete: boolean` (set true after merge succeeds)
- `deleteWorktree: boolean` (default true)
- `deleteBranch: boolean` (default true)
- `cleaning: boolean` (cleanup in progress)
- `cleanupError: string | null`

**Data fetching:**
- useEffect on `open`: when true, reset all state to defaults, then invoke `get_branch_diff_summary` with project.path, branchName, mergeTarget. Set diffData on success, handle error gracefully.

**Step navigation (per locked decisions):**
- Forward: "Next" button advances step. Step 2->3 only after merge. Step 3->4 only after mergeComplete.
- Back: "Back" ghost button on steps 1-3. Hidden on step 0. Hidden after mergeComplete (cannot undo merge). Hidden during merge execution.
- No skip-forward allowed.

**Footer button logic (per UI spec copywriting):**
- Step 0: "Next" -> setStep(1)
- Step 1: "Next" -> setStep(2)
- Step 2 pre-merge: "Merge Branch" button (triggers merge in MergeStep via ref or callback)
- Step 2 post-success: "Next" -> setStep(3)
- Step 3: "Clean Up & Close" if either checkbox on; "Close" if both off

**Merge step integration:**
- Pass `onSuccess={() => setMergeComplete(true)}` to MergeStep
- Read merge store step to determine if executing (for dialog close prevention)
- Prevent dialog close (escape, overlay click) while merge is executing: use `onPointerDownOutside={(e) => e.preventDefault()}` and `onEscapeKeyDown={(e) => e.preventDefault()}` on DialogContent when merge is in progress

**Cleanup handler:**
- When user clicks "Clean Up & Close": set cleaning=true, invoke `delete_worktree` with `{ projectPath: project.path, worktreePath: tab.worktreePath, branchName, deleteWorktree, deleteBranch }`
- On success: close dialog via onOpenChange(false), then setTimeout 200ms to call `useTerminalStore.getState().closeTab(tab.id)` (per pitfall 4: avoid race condition with dialog animation)
- On failure: set cleanupError with message, keep dialog open for retry
- If both checkboxes unchecked and user clicks "Close": just close dialog, no cleanup, no tab close

**Dialog shell (per UI spec):**
- Width: max-w-[520px]
- Background: bg-[var(--grove-deep)] border border-[var(--grove-canopy)]
- DialogHeader: title "Post-Session Review", description "{branchName} into {mergeTarget}" in grove-stone
- WizardStepper below header with 24px gap
- Step content area with min-h-[200px] and ScrollArea max-h-[360px]
- Separator before footer
- Footer: left side "Back" ghost button, right side primary action button

**Important:** Call `useMergeStore.getState().clearOperation()` when dialog opens to prevent stale merge state from a previous operation (per pitfall 1: singleton store).

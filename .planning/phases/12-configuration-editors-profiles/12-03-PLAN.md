1. Create `src-ui/src/components/config/SettingsJsonEditor.tsx`:
   - Props: `projectPath: string`.
   - File path: `projectPath + '/.claude/settings.json'`.
   - Load content via `useFileEditor`. Parse JSON on load to populate structured form.
   - **Two modes** toggled by a tab bar at top: "Form" and "JSON".

   **Form mode** (default):
   - Parse the loaded JSON string into a typed structure. The `.claude/settings.json` schema has these known sections:
     - `permissions`: object with `allow` and `deny` arrays of permission strings (e.g., "Bash(git:*)", "Read(~)")
     - `hooks`: object with `preToolUse` and `postToolUse` arrays of hook objects `{ matcher, command }`
     - `mcpServers`: object mapping server name to `{ command, args, env }` config
   - Render each section as a collapsible card:
     - **Permissions**: Two lists (Allow, Deny). Each item is an editable text input. "Add" button appends empty row. "X" button removes row.
     - **Hooks**: Two lists (Pre-tool, Post-tool). Each hook shows matcher and command inputs side-by-side. Add/remove buttons.
     - **MCP Servers**: List of server cards. Each shows name (editable), command, args (comma-separated), env vars (key=value pairs). Add server / remove server buttons.
   - Form changes immediately serialize back to JSON string and update the useFileEditor content (keeping it in sync for dirty tracking).

   **JSON mode**:
   - Render `@uiw/react-codemirror` with `json()` extension and `groveEditorTheme`.
   - Direct text editing of the raw JSON.
   - On switching back to Form mode, re-parse JSON. If parse fails, show error banner and stay in JSON mode.

   **Validation (CONF-06)**:
   - Before save: attempt `JSON.parse(content)`. If it fails, show red error banner with parse error message and line number. Disable save button.
   - In Form mode, validation is implicit (form always produces valid JSON).

   **Save**: toolbar save button, disabled when not dirty or validation fails. Calls `save()` from useFileEditor.

   **New file**: If settings.json doesn't exist, show "No settings.json found" with a "Create" button that writes a default `{ "permissions": { "allow": [], "deny": [] } }` template.
  </action>

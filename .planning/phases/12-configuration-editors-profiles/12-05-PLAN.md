<task type="auto">
  <name>Task 2: Profile environment injection in terminal spawn</name>
  <files>src-tauri/src/terminal/commands.rs, src-tauri/src/terminal/pty.rs</files>
  <read_first>src-tauri/src/terminal/commands.rs, src-tauri/src/terminal/pty.rs, src-tauri/src/config/models.rs, src-tauri/src/config/persistence.rs</read_first>
  <action>
1. Modify `src-tauri/src/terminal/commands.rs` `terminal_spawn` function:
   - Add a new optional parameter: `project_id: Option<String>`.
   - After UNC path resolution, before PTY spawn, look up the project's profile:
     ```rust
     let env_overrides: HashMap<String, String> = if let Some(ref pid) = project_id {
         if let Ok(config) = crate::config::persistence::load_or_create_config(&app_handle) {
             if let Some(project) = config.projects.iter().find(|p| p.id == *pid) {
                 if let Some(ref profile_id) = project.profile_id {
                     config.profiles.iter()
                         .find(|p| p.id == *profile_id)
                         .map(|p| p.env_vars.clone())
                         .unwrap_or_default()
                 } else {
                     // No explicit profile — use default profile if one exists (PROF-05)
                     config.profiles.iter()
                         .find(|p| p.is_default)
                         .map(|p| p.env_vars.clone())
                         .unwrap_or_default()
                 }
             } else {
                 HashMap::new()
             }
         } else {
             HashMap::new()
         }
     } else {
         HashMap::new()
     };
     ```
   - Pass `env_overrides` to `pty::spawn_pty`.

2. Modify `src-tauri/src/terminal/pty.rs` `spawn_pty` function:
   - Add parameter: `env_overrides: std::collections::HashMap<String, String>`.
   - After the existing `cmd.env("COLORTERM", ...)` line, iterate and apply:
     ```rust
     for (key, value) in &env_overrides {
         cmd.env(key, value);
     }
     ```
   - This ensures profile env vars are injected into the PTY process before spawn.

3. Update the call site in `terminal_spawn` to pass `env_overrides`:
   - Change `pty::spawn_pty(&resolved, cols, rows, on_event, app_handle)` to `pty::spawn_pty(&resolved, cols, rows, on_event, app_handle, env_overrides)`.

4. NOTE: The frontend `terminalStore` that calls `terminal_spawn` may need to pass `project_id`. The executor should check `src-ui/src/stores/terminal-store.ts` for the `spawnTerminal` function and add `projectId?: string` parameter, passing it to the Tauri invoke call. If project_id is already available in the terminal store context, use it. Otherwise add it as an optional parameter.

## Interaction Contract

### Toast Stack Behavior

1. New toasts enter from bottom-right, sliding up with 200ms ease-out animation (Sonner default)
2. Maximum 3 toasts visible simultaneously
3. Priority ordering: error > waiting > info. When at capacity, new info toasts replace oldest info toast. Error toasts are never auto-replaced.
4. Error toasts persist until user clicks the X dismiss button
5. Informational toasts auto-dismiss after 5 seconds with a subtle opacity fade-out (300ms)
6. Hovering over any toast in the stack pauses ALL auto-dismiss timers (Sonner default behavior)

### Toast Action: Navigate to Session

1. Each session-related toast includes a "View Session" action button
2. Clicking the action button: switches to the relevant session tab in SessionManager and dismisses the toast
3. Action button is right-aligned within the toast, styled as text link in state color

### OS Notification Coordination

1. When app window IS focused: fire toast only (no OS notification)
2. When app window is NOT focused: fire both toast AND OS notification (chime + taskbar flash via existing `fireWaitingAlert()`)
3. Never show both an OS notification and a toast for the same event when the app is in the foreground

### Toast Content Templates

| Event | Title | Description | Action |
|-------|-------|-------------|--------|
| Session waiting | "{branch} needs input" | "Session is waiting for your response" | "View Session" |
| Session error | "{branch} error" | "Session encountered an error" | "View Session" |
| Session idle | "{branch} idle" | "Session has gone idle" | "View Session" |
| Merge success | "Merge complete" | "{branch} merged into {target}" | none |
| Merge failure | "Merge failed" | "{branch}: {error summary}" | none |

Output: Job Object wrapper module integrated into TerminalSession spawn and kill flow.

  <action>
1. Add `windows-sys` dependency to Cargo.toml under `[target.'cfg(windows)'.dependencies]`:
   ```toml
   [target.'cfg(windows)'.dependencies]
   windows-sys = { version = "0.59", features = ["Win32_System_JobObjects", "Win32_Foundation", "Win32_Security"] }
   ```
   Use version 0.59 (latest stable). The features needed are:
   - `Win32_System_JobObjects` for CreateJobObjectW, AssignProcessToJobObject, SetInformationJobObject
   - `Win32_Foundation` for CloseHandle, HANDLE, BOOL
   - `Win32_Security` for security attributes (null)

2. Create `src-tauri/src/terminal/job_object.rs` with three public functions:
   - `create_job_object() -> Result<isize, String>` -- Creates a nameless Job Object with `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE` flag set. Uses `CreateJobObjectW(null, null)`, then `SetInformationJobObject` with `JOBOBJECT_EXTENDED_LIMIT_INFORMATION` struct where `BasicLimitInformation.LimitFlags = JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE`. Returns the HANDLE as isize.
   - `assign_process_to_job(job_handle: isize, pid: u32) -> Result<(), String>` -- Opens the process with `PROCESS_SET_QUOTA | PROCESS_TERMINATE` via `OpenProcess`, then calls `AssignProcessToJobObject`. Closes the process handle after assignment.
   - `close_job_object(job_handle: isize)` -- Calls `CloseHandle` on the job handle. This triggers KILL_ON_JOB_CLOSE, terminating all processes in the job.

   All functions are `#[cfg(windows)]` gated. Use `unsafe` blocks with clear safety comments. Handle null/invalid handle returns as errors.

3. Update `src-tauri/src/terminal/mod.rs`:
   - Add `pub mod job_object;` declaration
   - Add `pub(crate) job_handle: Option<isize>` field to `TerminalSession` struct (Option because non-Windows builds won't have it)
   - Make `TerminalSession` implement `Drop` that calls `close_job_object` if `job_handle.is_some()` -- this ensures cleanup even if kill() isn't explicitly called (e.g., app crash recovery)

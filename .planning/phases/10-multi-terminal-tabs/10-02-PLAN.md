1. **Rewrite `src-ui/src/stores/terminal-store.ts`** with Map-based multi-session state:

   Define a `TerminalTab` interface:
   ```typescript
   interface TerminalTab {
     id: string;           // terminal ID from Rust PTY spawn (or 'pending-{uuid}' before spawn completes)
     worktreePath: string;
     branchName: string;
     isConnected: boolean;
     createdAt: number;    // Date.now() for duration calculation (TERM-03)
   }
   ```

   New store shape:
   ```typescript
   interface TerminalStoreState {
     tabs: Map<string, TerminalTab>;   // keyed by tab ID (= terminal ID)
     activeTabId: string | null;
     // Derived
     hasAnyTabs: () => boolean;
     // Actions
     addTab: (worktreePath: string, branchName: string) => string;  // returns pending tab ID
     activateTab: (tabId: string, terminalId: string) => void;      // promotes pending -> real ID
     switchTab: (tabId: string) => void;
     closeTab: (tabId: string) => void;
     setTabConnected: (tabId: string, connected: boolean) => void;
     getTabForWorktree: (worktreePath: string) => TerminalTab | undefined;
   }
   ```

   Implementation details:
   - `addTab`: Creates a TerminalTab with a pending ID (`pending-${crypto.randomUUID()}`), inserts into Map, sets as activeTabId. Returns the pending ID. This is called by Dashboard when user clicks Launch.
   - `activateTab`: Called by TerminalInstance after spawn completes. Removes the pending entry, inserts with real terminal ID, updates activeTabId to the real ID.
   - `switchTab`: Sets activeTabId. Does NOT destroy anything.
   - `closeTab`: Removes tab from Map. If it was active, switch to the most recently created remaining tab (or null if empty).
   - `getTabForWorktree`: Finds a tab by worktreePath. Used by Dashboard to prevent duplicate tabs.
   - `hasAnyTabs`: Returns `tabs.size > 0`. Used by Dashboard for conditional split-pane rendering.

   IMPORTANT: Use `immer` middleware or manual Map cloning for Zustand reactivity. Zustand doesn't detect Map mutations. Pattern: `set({ tabs: new Map(get().tabs) })` after every mutation.

Simplest approach: add `refit: () => void` to useTerminal return that calls `fitRef.current?.fit()` inside a rAF.

2. **Update `useTerminal` hook** to expose a `refit` function:
   Add to the return object:
   ```typescript
   const refit = useCallback(() => {
     requestAnimationFrame(() => {
       if (fitRef.current && termRef.current) {
         fitRef.current.fit();
       }
     });
   }, []);
   return { terminal: termRef, write, refit };
   ```
   This is called by TerminalInstance when a tab becomes visible again.

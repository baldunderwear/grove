    **src-ui/src/types/merge.ts:**
    Create TypeScript interfaces matching Rust structs EXACTLY (snake_case, per Phase 02 decision):
    - `CommitInfo` — { oid: string, message: string, author: string, timestamp: number }
    - `ChangelogFragment` — { path: string, name: string, is_legacy: boolean }
    - `MergePreview` — { source_branch: string, target_branch: string, commits_to_merge: CommitInfo[], changelog_fragments: ChangelogFragment[], current_build: number | null, next_build: number | null, can_fast_forward: boolean, has_conflicts: boolean }
    - `MergeResult` — { success: boolean, new_build: number | null, commits_merged: number, changelog_renames: [string, string][], warnings: string[] }
    - `MergeHistoryEntry` — { source_branch: string, target_branch: string, result: MergeResult, timestamp: number }

    **src-ui/src/stores/merge-store.ts:**
    Create a Zustand store following the branch-store.ts pattern:
    - Module-level `let fetchCounter = 0;` for race protection
    - State: `preview: MergePreview | null`, `result: MergeResult | null`, `loading: boolean`, `error: string | null`, `step: MergeStep` (type alias: `'idle' | 'preview' | 'confirm' | 'executing' | 'summary' | 'error'`), `history: MergeHistoryEntry[]`
    - `fetchPreview(projectPath, sourceBranch, mergeTarget, buildPatterns, changelogConfig)` — increments fetchCounter, sets loading true, invokes `merge_preview` with camelCase params, sets preview + step='preview'. On error: sets error + step='error'. Checks fetchId !== fetchCounter for race protection.
    - `executeMerge(projectPath, sourceBranch, mergeTarget, buildPatterns, changelogConfig)` — sets step='executing', invokes `merge_branch` with camelCase params. On success: sets result, adds to history (prepend, cap at 50 entries via `.slice(0, 50)`), sets step='summary'. On error: sets error + step='error'.
    - `clearOperation()` — resets preview, result, error to null, step to 'idle'. Does NOT clear history.
    - `clearHistory()` — empties history array.
    - Export `MergeStep` type from store file.
    - Per FR-04.8: NO push commands anywhere. Add a comment at the top: "// Merge is local only -- never pushes to remote (FR-04.8)"
    - Import invoke from `@tauri-apps/api/core`, types from `@/types/merge`, config types from `@/types/config`.
  </action>

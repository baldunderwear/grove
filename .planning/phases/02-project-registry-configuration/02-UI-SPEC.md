## Layout Contract

### Overall Structure

```
+------------------+-------------------------------------+
|                  |                                     |
|  Sidebar (280px) |  Main Content (flex-1)              |
|  fixed height    |  scrollable                         |
|                  |                                     |
|  [Grove title]   |  [Section heading]                  |
|  [Add Project]   |  [Config form / Settings / Empty]   |
|  [Project list]  |                                     |
|  ...             |                                     |
|  [Settings gear] |                                     |
|                  |                                     |
+------------------+-------------------------------------+
```

### Sidebar

- Width: 280px, fixed, non-collapsible
- Background: gray-900
- Top: "Grove" heading (20px semibold) + "Add Project" button (full width, emerald-500 bg)
- Middle: Scrollable project list. Each item is 40px tall with 8px vertical padding.
- Item layout: health dot (8px circle, 4px gap) + project name (14px, truncate with ellipsis)
- Selected item: gray-800 background, white text
- Hover item: gray-850 background
- Bottom: Settings gear icon (lucide `Settings`), 16px padding from bottom edge

### Main Content — Project Config Panel

Shown when a project is selected in sidebar.

- Header: Project name (20px semibold) + path below (12px, gray-400, truncate)
- Health status badge: green dot "Healthy" or red dot "Path not found" / "Not a git repo"
- Form sections separated by 24px gap:

**Section: Merge Settings**
- Merge target branch (Input, placeholder: "main")
- Branch prefix (Input, placeholder: "wt/")

**Section: Build Files** (optional)
- Build file patterns list (each as Input with remove icon)
- "Add pattern" text button below list
- Helper text: "Glob patterns for files containing build numbers. Leave empty for plain merge."

**Section: Changelog** (optional)
- Changelog directory (Input, placeholder: "docs/changelog")
- Fragment naming pattern (Input, placeholder: "worktree-{name}.md")
- Helper text: "Leave empty to skip changelog handling during merge."

**Section: Danger Zone**
- "Remove Project" button (red-500 outline variant, right-aligned)

### Main Content — Settings Page

Shown when Settings is clicked in sidebar.

- Header: "Settings" (20px semibold)
- Form sections:

**Section: General**
- Refresh interval: Input (number) + "seconds" label, default 30
- Start minimized: Toggle/checkbox
- Start with Windows: Toggle/checkbox

### Main Content — Empty State

Shown when no projects registered.

- Centered vertically and horizontally in main content
- Folder icon (lucide `FolderOpen`, 48px, gray-500)
- Heading: "No projects yet" (20px semibold, gray-300)
- Body: "Add a git repository to start managing worktrees." (14px, gray-400)
- "Add Project" button (emerald-500, same style as sidebar button)

### Interactions

| Action | Behavior |
|--------|----------|
| Click "Add Project" | Opens native OS directory picker via Tauri dialog API |
| Select directory | Validate: is git repo? If yes, add to list + select it. If no, show inline error. |
| Click project in sidebar | Load project config in main panel |
| Edit any form field | Auto-save on blur via Zustand store -> Tauri `save_config` command |
| Click "Remove Project" | Opens confirmation dialog before removal |
| Click Settings gear | Show settings page in main content area |
| Invalid directory selected | Show error inline below "Add Project" button in sidebar |

## State Inventory

| State | Visual Treatment |
|-------|-----------------|
| No projects registered | Empty state (centered icon + text + CTA) |
| Project selected, healthy | Config form shown, green health dot |
| Project selected, unhealthy | Config form shown (fields may be read-only), red health dot + status text |
| Directory picker open | Native OS dialog (no custom UI needed) |
| Invalid directory selected | Inline error text below "Add Project" button, red-500 text, 12px |
| Remove confirmation open | Dialog overlay with gray-950/80 backdrop |
| Settings page active | No project selected highlight in sidebar, settings gear highlighted |
| Field saving (blur) | Brief emerald-500/30 border flash (200ms transition) |

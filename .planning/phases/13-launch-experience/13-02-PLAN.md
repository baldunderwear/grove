    **TemplateManager.tsx** — Template CRUD UI component:
    - Props: `{ onSelect: (template: PromptTemplate) => void }`.
    - Reads templates from `useConfigStore(s => s.config?.templates ?? [])`.
    - Displays template list with name and first line of body as preview.
    - "New Template" button opens inline form: name input + body textarea.
    - Each template row has Edit (pencil icon) and Delete (trash icon) buttons.
    - Edit opens same inline form pre-filled. Save calls `updateTemplate(id, { name, body })`.
    - Create calls `addTemplate(name, body)`. Delete calls `removeTemplate(id)` with confirmation.
    - Click on a template row calls `onSelect(template)`.
    - Use existing shadcn components: Button, Input, ScrollArea. Use lucide icons: Plus, Pencil, Trash2.
    - Style with Tailwind using Grove design tokens (var(--grove-*) colors).

    **ContextFilePicker.tsx** — File selector component:
    - Props: `{ worktreePath: string; selectedFiles: string[]; onSelectionChange: (files: string[]) => void }`.
    - On mount, call `invoke<DirEntry[]>('list_directory', { path: worktreePath })` to get file listing.
    - Define DirEntry type locally: `{ name: string; path: string; is_dir: boolean; size: number; modified: number }`.
    - Display as a flat list of files with checkboxes (NOT a tree view — keep it simple per Claude's discretion).
    - Filter out hidden files (starting with `.`) and common non-useful dirs (`node_modules`, `target`, `.git`).
    - Each row shows filename, file size in human-readable form.
    - Clicking a directory navigates into it (update path state, re-invoke list_directory). Show breadcrumb path for navigation back.
    - Checkbox toggles file in/out of `selectedFiles`. Parent is notified via `onSelectionChange`.
    - Use ScrollArea for the file list. Max height ~200px.

    **LaunchDialog.tsx** — Main launch dialog:
    - Props: `{ open: boolean; onOpenChange: (open: boolean) => void; worktreePath: string; branchName: string; projectId?: string; projectPath: string }`.
    - Uses shadcn Dialog component (Dialog, DialogContent, DialogHeader, DialogTitle, DialogFooter).
    - Layout has three sections stacked vertically:
      1. **Template section**: Dropdown/select to pick a saved template OR "Custom prompt" option. When template selected, populate the prompt textarea with template body, substituting `{branch}` with `branchName`, `{project}` with project name (derive from projectPath), `{path}` with worktreePath. Button to open TemplateManager in a sub-dialog for CRUD.
      2. **Prompt textarea**: Editable textarea showing the (possibly template-derived) prompt. User can modify before launch. Placeholder: "Enter your prompt for Claude Code...".
      3. **Context files section**: Collapsible section "Add context files" using ContextFilePicker. Shows count of selected files as badge.
    - Footer has "Cancel" and "Launch" buttons. Launch button disabled if prompt is empty.
    - On Launch click:
      1. Get prompt text (with variables already substituted).
      2. Get selected context file paths.
      3. Call `addTab(worktreePath, branchName, projectId)` to create terminal tab.
      4. Store the pending prompt + context files in terminal store (see Task 2 for store changes) so TerminalPanel can auto-send after connection.
    - Dialog width: `max-w-2xl`.

    **Terminal store changes** (src-ui/src/stores/terminal-store.ts):
    1. Add to `TerminalTab` interface:
       ```typescript
       initialPrompt?: string;      // Prompt to auto-send after Claude Code ready
       contextFiles?: string[];     // File paths to pass as --file flags (future: modify terminal_spawn)
       ```
    2. Update `addTab` signature to accept optional launch options:
       ```typescript
       addTab: (worktreePath: string, branchName: string, projectId?: string, launchOptions?: { prompt?: string; contextFiles?: string[] }) => string;
       ```
       In the implementation, spread launchOptions into the tab: `initialPrompt: launchOptions?.prompt, contextFiles: launchOptions?.contextFiles`.
    3. Add `clearInitialPrompt` action:
       ```typescript
       clearInitialPrompt: (tabId: string) => void;
       ```
       Sets `initialPrompt` to `undefined` for the given tab (so it only auto-sends once).

    **TerminalPanel auto-send** (src-ui/src/components/terminal/TerminalPanel.tsx):
    In `TerminalInstance`, after the terminal connects and `activateTab` is called (the existing `onEvent` handler where `type === 'Data'` fires first time or after a brief delay):
    - Read the tab's `initialPrompt` from store.
    - If `initialPrompt` exists, set up a one-time auto-send:
      - Wait for the terminal to be "ready" — use a simple 2-second delay after the first Data event (Claude Code prints its banner). This avoids complex ready-state detection.
      - After delay, call `invoke('terminal_write', { terminalId, data: initialPrompt + '\n' })`.
      - Call `clearInitialPrompt(tabId)` immediately so it never sends twice.
    - Use a ref (`autoSendDoneRef`) to ensure the auto-send only fires once even if re-renders occur.

    **Dashboard changes** (src-ui/src/pages/Dashboard.tsx):
    1. Add state: `const [launchBranch, setLaunchBranch] = useState<BranchInfo | null>(null);`
    2. Change `handleLaunch` to:
       - If existing tab for worktree, switch to it (existing behavior).
       - Else, set `setLaunchBranch(branch)` to open the LaunchDialog instead of directly calling addTab.
    3. Add `handleLaunchConfirm` callback:
       ```typescript
       const handleLaunchConfirm = (prompt: string, contextFiles: string[]) => {
         if (!launchBranch || !project) return;
         addTab(launchBranch.worktree_path, launchBranch.name, selectedProjectId ?? undefined, {
           prompt: prompt || undefined,
           contextFiles: contextFiles.length > 0 ? contextFiles : undefined,
         });
         setLaunchBranch(null);
       };
       ```
    4. Import and render `LaunchDialog`:
       ```tsx
       <LaunchDialog
         open={!!launchBranch}
         onOpenChange={(open) => { if (!open) setLaunchBranch(null); }}
         worktreePath={launchBranch?.worktree_path ?? ''}
         branchName={launchBranch?.name ?? ''}
         projectId={selectedProjectId ?? undefined}
         projectPath={project.path}
         onLaunch={handleLaunchConfirm}
       />
       ```
       Note: Add `onLaunch` as a callback prop to LaunchDialog (update Task 1's LaunchDialog to call `onLaunch(prompt, contextFiles)` instead of directly calling addTab — this keeps the dialog reusable and the Dashboard as the coordinator).

    **BranchTable** — no changes needed. The `onLaunch` callback already passes `BranchInfo` to Dashboard.
